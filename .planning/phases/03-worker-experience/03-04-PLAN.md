**1. Create `src/actions/assignments.ts`:**
- `'use server'` directive
- Import `createClient` from `@/lib/supabase/server`, `z` from `zod`
- Export `async function assignSopToRole(sopId: string, role: string)`:
  - Validate role is one of: 'worker', 'supervisor', 'admin', 'safety_manager' using Zod
  - Get supabase client, get current user, get user's org_id from `user_profiles`
  - Verify user role is 'admin' or 'safety_manager' (guard)
  - Insert into `sop_assignments`: `{ organisation_id, sop_id: sopId, assignment_type: 'role', role, assigned_by: user.id }`
  - Return `{ success: true, id: data.id }` or `{ success: false, error: string }`
- Export `async function assignSopToUser(sopId: string, userId: string)`:
  - Get supabase client, verify admin role
  - Insert into `sop_assignments`: `{ organisation_id, sop_id: sopId, assignment_type: 'individual', user_id: userId, assigned_by: currentUser.id }`
  - Return `{ success: true, id: data.id }` or `{ success: false, error: string }`
- Export `async function removeAssignment(assignmentId: string)`:
  - Get supabase client, verify admin role
  - Delete from `sop_assignments` where `id = assignmentId`
  - Return `{ success: true }` or `{ success: false, error: string }`
- Export `async function getAssignments(sopId: string)`:
  - Get supabase client
  - Select from `sop_assignments` where `sop_id = sopId`, order by `created_at`
  - Return `{ success: true, assignments: data }` or `{ success: false, error: string }`
- Export `async function getOrgMembers()`:
  - Get supabase client, get user's org
  - Select from `user_profiles` where `organisation_id` matches, select `id, full_name, role, email` (join with auth.users for email if needed, or use user_profiles fields)
  - Return `{ success: true, members: data }`

**2. Create `src/app/api/sops/[sopId]/assignments/route.ts`:**
- POST handler: calls `assignSopToRole` or `assignSopToUser` based on request body `assignment_type`
- DELETE handler: calls `removeAssignment` based on `assignmentId` in request body
- GET handler: calls `getAssignments` with sopId from params
- All handlers return NextResponse.json with appropriate status codes (200, 201, 400, 403)
- Validate request body with Zod schemas
