Convention:
- One `test.describe` per requirement group (e.g. SB-SECT)
- One or more `test.fixme` per requirement ID — the test title starts with the
  requirement ID so CI can grep for coverage (e.g. `test.fixme('SB-SECT-01 ...')`)
- The `fixme` empty-body signals "stub; fill in later phase"
- Playwright config uses `testMatch` regex to group them into a project
