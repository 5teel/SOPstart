## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (`gap-1`, `px-1`) |
| sm | 8px | Compact element spacing (`gap-2`, `p-2`) |
| md | 16px | Default element spacing (`px-4`, `gap-4`) |
| lg | 24px | Section padding (`p-6`, `py-6`) |
| xl | 32px | Layout gaps (`py-8`) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- **Tap targets**: minimum 72px height on all primary action buttons (`h-[72px]`) ŌĆö project-wide `--min-tap-target: 72px` custom property per WORK-09. Applied to: primary CTA buttons, format selection confirm, upload triggers.
- **Secondary CTAs and nav actions**: minimum 44px height (`min-h-[44px]`) ŌĆö for actions that are not the primary touch target on a given screen.
- **Sticky header bar**: fixed 56px height (`h-[56px]`) ŌĆö matches existing `VideoGeneratePanel` header pattern.
- **Stepper connector lines**: 1px height (`h-px`) ŌĆö visual only, not a touch target.

Source: `UploadDropzone.tsx` buttons all use `h-[72px]`. `ReviewClient.tsx` secondary actions use `min-h-[44px]`. `VideoGeneratePanel.tsx` header uses `h-[56px]`.

## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm` |
| Label / Button | 16px | 600 (semibold) | 1.25 | `text-base font-semibold` |
| Heading | 18px | 600 (semibold) | 1.2 | `text-lg font-semibold` (or `text-base font-semibold` for sub-headings) |
| Display / Page title | 20px | 700 (bold) | 1.2 | `text-xl font-bold` |

Additional type uses observed in pattern components:
- Stepper stage labels: 12px / 400 (`text-xs`) ŌĆö active stage gains `font-semibold`
- Caption / metadata: 12px / 400 (`text-xs text-steel-400` or `text-steel-500`)
- Large CTA button text: 18px / 700 (`text-lg font-bold`) ŌĆö used on primary upload/generate CTAs

Source: `UploadDropzone.tsx`, `VideoGeneratePanel.tsx`, `ParseJobStatus.tsx`, `VideoGenerationStatus.tsx`.

## Color

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `steel-900` | `#111827` | Page backgrounds, modal backdrops |
| Secondary (30%) | `steel-800` | `#1f2937` | Cards, panels, input fields, queue items, stepper containers |
| Tertiary surface | `steel-700` | `#374151` | Borders, secondary buttons, inactive tab underlines, dividers |
| Muted text | `steel-400` | `#9ca3af` | Secondary labels, placeholder text, metadata, inactive stepper stages |
| Accent (10%) | `brand-yellow` | `#f59e0b` | See reserved-for list below |
| Warning / Human gate | `brand-orange` | `#ea580c` | OCR warnings, review-required checkpoint, video gen failure states |
| Destructive | `red-400` | Tailwind red-400 | Delete buttons and error text only (inline, not full backgrounds) |
| Success indicator | `green-400` | Tailwind green-400 | Completed stage labels, CheckCircle icons, success banners |
| In-progress indicator | `blue-400` | Tailwind blue-400 | Loader2 spinner during active processing stages |

**Accent (`brand-yellow`) reserved for:**
1. Primary CTA buttons (background fill: `bg-brand-yellow text-steel-900`)
2. Active tab underline in mode tab bar
3. Active stepper stage label text (`text-brand-yellow font-semibold`)
4. Stepper connector lines for completed stages (`bg-brand-yellow`)
5. Drag-over state on dropzone (`border-brand-yellow bg-brand-yellow/10`)
6. Selected radio option border and inner dot on format picker
7. Focus ring on text inputs (`focus:border-brand-yellow`)
8. Inline text links (`text-brand-yellow hover:text-amber-400`)

**NOT for:** general hover states, informational icons, background fills on cards, borders at rest.

**Human-gate checkpoint color:** `brand-orange` background tint (`bg-brand-orange/20 border border-brand-orange/50`) ŌĆö the "Review required" paused state on the progress page uses this, not an error red, because it is a designed human pause, not a failure.

Source: globals.css token definitions + pattern scan across all component files.

## Component Inventory

Components this phase creates or modifies:

### 1. `UploadDropzone.tsx` ŌĆö Modified
**Change:** Add "Generate video SOP" secondary entry to the Upload file tab panel.

**Visual pattern:** New button in the existing `flex gap-3 flex-wrap justify-center` button row inside the drop zone. Matches the `bg-steel-700 text-steel-100` secondary button style used by "Scan document" and "Browse video". Includes a `Film` or `Clapperboard` lucide icon (16├Ś16 or 20├Ś20, `shrink-0`).

**Format selection:** A modal (not inline segmented control). Clicking "Generate video SOP" opens a modal overlay (`fixed inset-0 z-50 flex items-center justify-center bg-black/70`). Modal inner container: `bg-steel-900 rounded-2xl p-6 max-w-lg w-full mx-4`. Modal contains:
- Heading: "Choose video format" (`text-base font-semibold text-steel-100 mb-4`)
- Format radio cards: reuse exact visual pattern from `VideoGeneratePanel.tsx` ŌĆö `flex items-start gap-4 p-4 rounded-xl border-2 cursor-pointer`, yellow border when selected
- "Start pipeline" confirm CTA: `h-[72px] w-full bg-brand-yellow text-steel-900 font-bold text-lg rounded-lg`
- Cancel / close: `X` icon top-right of modal, or a `bg-steel-700` secondary button

**Button position in dropzone:** Added to the button row after "Browse video". The button must have `h-[72px]` height to maintain consistent touch targets.

### 2. Pipeline Progress Page ŌĆö New route
**Route shape:** `/admin/sops/[sopId]/pipeline` (planner may adjust per D-06 state shape, but visual contract applies to either URL pattern).

**Layout:** Single-column, `max-w-2xl mx-auto px-4 py-8`. Sticky header bar: `h-[56px] sticky top-0 z-20 bg-steel-900 border-b border-steel-700 px-4 flex items-center gap-3` ŌĆö left: back arrow (`ArrowLeft` icon, `text-steel-400 hover:text-steel-100`), centre-left: SOP title (truncated, `text-sm font-medium text-steel-100`).

**Unified stepper:** Single horizontal stepper rendering all 5 named stages as one continuous pipeline. Stage list (in order):

| # | Stage key | Label | State when human gate |
|---|-----------|-------|----------------------|
| 1 | `uploading` | Uploading | Auto-progresses |
| 2 | `parsing` | Parsing | Auto-progresses |
| 3 | `review` | Review | **Paused** ŌĆö human gate |
| 4 | `generating` | Generating video | Auto-progresses after publish |
| 5 | `ready` | Ready | Terminal success |

**Stepper visual rules (matches `ParseJobStatus` + `VideoGenerationStatus` stepper):**
- Completed stage label: `text-xs text-green-400`
- Active stage label: `text-xs text-brand-yellow font-semibold` with `aria-current="step"`
- Pending stage label: `text-xs text-steel-600`
- Connector line completed: `h-px flex-1 min-w-[8px] bg-brand-yellow`
- Connector line pending: `h-px flex-1 min-w-[8px] bg-steel-700`
- Container: `flex items-center gap-1 overflow-x-auto` with `role="group" aria-label="Pipeline stages"`

**Stage 3 "Review required" paused state:** This is a human checkpoint, not an error. Render the stage body as:

```
bg-brand-orange/20 border border-brand-orange/50 rounded-xl p-5
  ClipboardCheck icon (24├Ś24, text-brand-orange)
  Heading: "Review required before video generates" (text-sm font-semibold text-steel-100)
  Body: "Check the parsed SOP, approve all sections, then publish to continue." (text-sm text-steel-400)
  CTA button: "Review SOP now ŌåÆ" (h-[72px] w-full bg-brand-yellow text-steel-900 font-bold text-lg rounded-lg)
    ŌåÆ deep-links to /admin/sops/[sopId]/review
```

The stepper connector between "Parsing" and "Review" uses `bg-brand-yellow` (parsing is complete). The connector between "Review" and "Generating video" uses `bg-steel-700` (not yet reached).

**Stage 4 "Generating video" active state:** Compose `VideoGenerationStatus` component (existing) below the stepper, showing the sub-stages (Analysing ŌåÆ Slides ŌåÆ Narration ŌåÆ Rendering ŌåÆ Ready).

**Stage 5 "Ready" terminal state:**

```
bg-green-500/20 border border-green-500/40 rounded-xl px-5 py-5
  CheckCircle icon (24├Ś24, text-green-400)
  Heading: "Video SOP ready" (text-sm font-semibold text-steel-100)
  Body: "Review the video and publish it when you're happy with the audio." (text-sm text-steel-400)
  CTA button: "Preview and publish video ŌåÆ" (h-[72px] w-full bg-brand-yellow text-steel-900 font-bold text-lg rounded-lg)
    ŌåÆ navigates to /admin/sops/[sopId]/video
```

**Polling:** Realtime + polling hybrid. Subscribe to `postgres_changes` on `parse_jobs` and `video_generation_jobs`. Fall back to 5s polling after 5s delay if no realtime event fires (matches Phase 2 + Phase 8 pattern).

### 3. Review page ŌĆö No structural changes
The existing `/admin/sops/[sopId]/review` page is not modified. A "Return to pipeline" back-link CTA is added contextually only when the admin arrived from the pipeline flow. This is a single back-arrow link in the review page header or a `ŌåÉ Back to pipeline` text link beneath the publish button. Uses existing `text-brand-yellow text-sm font-medium hover:text-amber-400` link style.

| State | Appearance |
|-------|-----------|

| Format selected | Selected card has `border-brand-yellow` border and filled radio dot (`bg-brand-yellow`). CTA is enabled. |

### Pipeline progress page ŌĆö stage body

| Stage | Body content |
|-------|-------------|
| Uploading (active) | `Loader2` blue-400 spinner + "Uploading your file..." label |
| Parsing (active) | `Loader2` blue-400 spinner + "Crunching your SOPŌĆ”" label + "This can take a few minutes." sub-label |
| Review (paused) | Orange-tint checkpoint panel with deep-link CTA (see above) |
| Generating video (active) | `VideoGenerationStatus` component (existing) |
| Ready | Green success panel with navigate-to-video CTA (see above) |

### Parse failure on progress page

When `parse_jobs.status = 'failed'` and the pipeline is still in the parse stages, render using the existing `ParseJobStatus` failed state visual (orange `AlertTriangle`, error message, "Try again" / "Delete" action links). The unified stepper marks stages 1ŌĆō2 with red-400 colour on the failed stage label.

### Video generation failure (D-05)

The SOP stays published. The progress page detects the `video_generation_jobs.status = 'failed'` state and renders:

```
bg-steel-800 border border-steel-700 rounded-xl p-5
  AlertTriangle icon (20├Ś20, text-brand-orange)
  "Video generation failed" (text-sm font-semibold text-steel-100)
  Error message (text-xs text-steel-400, clamped to 2 lines)
  "Go to video panel" CTA (h-[72px] w-full bg-brand-yellow text-steel-900 font-bold rounded-lg)
    ŌåÆ navigates to /admin/sops/[sopId]/video (existing page with retry button per D-05)
```

## Accessibility Notes

- All stepper groups use `role="group"` with `aria-label` (matches existing pattern).
- Active stepper stage uses `aria-current="step"`.
- Format radio cards use `<fieldset>` + `<legend className="sr-only">` (matches existing `VideoGeneratePanel` pattern).
- Format modal uses a visually hidden label on the close button: `aria-label="Close"`.
- Review gate CTA and Ready CTA both meet 72px touch target minimum.
- Error messages use `role="alert"` where inline (matches existing `YoutubeError` pattern).
- Progress page stepper container: `overflow-x-auto` to support small screens without horizontal scroll on the page.
