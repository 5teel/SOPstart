## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Supabase Json type incompatibility with FlowGraph on JSONB update**
- **Found during:** Task 1 TypeScript check
- **Issue:** `supabase.from('sops').update({ flow_graph: parsed.data.graph })` raised TS2322 — Supabase's generated `Json` type requires an index signature `{ [key: string]: Json }` but `FlowGraph.nodes[]` contains optional fields (`blockProps?: Record<string, unknown>`) which TypeScript cannot prove satisfy `Json | undefined`. Same pattern that bit Phase 06 and Phase 09.
- **Fix:** Cast `parsed.data.graph as unknown as Json` — consistent with existing pattern throughout codebase.
- **Files modified:** src/actions/flow-graph.ts
- **Commit:** `6111ea9`

**2. [Rule 1 - Bug] updateSopFlowGraph result.error type was `string | undefined` not `string`**
- **Found during:** Task 1 TypeScript check (line 163 of flow-graph-field.tsx)
- **Issue:** `setError(result.error)` — `setError` is `Dispatch<SetStateAction<string | null>>` but `result.error` was inferred as `string | undefined` (Zod issue message can be undefined). TypeScript rejected the assignment.
- **Fix:** Changed to `setError(result.error ?? 'Unknown error')` — provides a non-null fallback.
- **Files modified:** src/lib/builder/flow-graph-field.tsx
- **Commit:** `6111ea9`

## FlowGraphEditor Architecture

- **Left palette:** 7 buttons (step, measurement, decision, escalate, signoff, inspect, zone) with accent-colour left border. Click adds a new node at `{x: 0, y: nodes.length * 100}`.
- **SVG canvas:** viewBox computed from node positions with min 800×600. Nodes are `<rect>` (160×48px) with accent fill at 18% opacity and accent stroke. Labels truncated at 22 chars.
- **Drag:** `onMouseDown` captures offset in SVG coordinate space via `getScreenCTM().inverse()`; `onMouseMove` snaps to 20px grid via `Math.round(v/20)*20`.
- **Edge mode:** toolbar toggle; click source then target — creates `{kind: 'sequential'}` edge. Edge kind editable in right panel when edge selected. Edge colour: green=yes, red=no, amber=escalate, grey=sequential.
- **Keyboard:** `Delete`/`Backspace` removes selected node (+ all its edges) or selected edge. Input focus guard prevents firing while typing in props panel.
- **Save:** calls `updateSopFlowGraph({ sopId, graph: { version: 1, nodes, edges } })` directly.

## CommandPalette + Ask Endpoint Architecture

- **JUMP TO STEP:** `sop.sop_sections.flatMap(s => s.sop_steps)` → up to 20 items. `onSelect` pushes `?tab=walkthrough&step={id}`.
- **ASK AI:** inline `<input>` with `onKeyDown stopPropagation` (prevents cmdk search filter from intercepting typing). Enter or Ask button POSTs to `/api/sops/{sopId}/ask`. Streams response via `ReadableStream.getReader()`.
- **TOOLS & HAZARDS:** sections filtered by `section_type` containing hazard/ppe/tools/zone/emergency OR any step with `warning` or `caution`. Tab destination is `hazards` for hazard/ppe sections, `tools` for others.
- **Auth gate:** `/api/sops/[sopId]/ask` calls `supabase.auth.getUser()` → 401 if unauthenticated. RLS ensures workers can only ask about their org's SOPs.
- **Context budget:** SOP context string capped at 12000 chars. Query capped at 500 chars. `claude-haiku-4-5` at 512 max_tokens for fast on-site responses.
