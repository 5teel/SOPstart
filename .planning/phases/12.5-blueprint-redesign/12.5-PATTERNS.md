**Mapped:** 2026-04-24

## Summary

Most of Phase 12.5 is pattern reuse: migrations follow `00020_section_layout_data.sql` verbatim; server actions follow `updateSectionLayout` (safeParse → JWT role guard → LWW guard → update → `{success: true} | {error}`); Dexie v5 follows the cumulative-stores bump pattern from v3→v4; offline flush loops copy `flushPhotoQueue`/`flushCompletions`; block components copy the named-function + co-located Zod schema export pattern from `HazardCardBlock.tsx`; the three-place contract is already comment-documented at `src/lib/builder/puck-config.tsx:399-418`. The tab shell refactor of `/sops/[sopId]/page.tsx` largely copies the current chrome (header + tabs + content) and swaps `SopSectionTabs` for a new `SopTabNav` with URL-driven state. Genuinely new surfaces: (1) the `VoiceCaptureControl` MediaRecorder → WebSocket → Deepgram state machine; (2) the `/api/voice/token` ephemeral-token mint; (3) the hand-rolled SVG `FlowGraphEditor` — each lacks an in-repo analog and must be authored from RESEARCH.md contracts.

## Patterns genuinely new to this phase

These surfaces have no in-repo analog and must be authored from RESEARCH.md contracts rather than copied:

1. **Deepgram ephemeral-token mint (`/api/voice/token`)** — Next.js route handler POSTing to `https://api.deepgram.com/v1/auth/grant`; 30s token TTL; auth-scoped. No prior project has called an external streaming-STT API. Follow RESEARCH.md §1 code block verbatim.

2. **Browser WebSocket streaming to Deepgram (`useDeepgramWebSocket`, `deepgram-stream.ts`)** — browsers forbid custom WS headers, so auth goes via the `Sec-WebSocket-Protocol: ['token', access_token]` subprotocol trick (Pitfall 3). Keepalive every 3s (Pitfall 4). `Finalize` + `CloseStream` control frames. No in-repo analog.

3. **MediaRecorder state machine (`VoiceCaptureControl`)** — `useReducer` with 5 discriminated-union states; feature-detect chain for mime types (iOS Safari mp4 fallback per Pitfall 6 + 9); feeds blob to Deepgram live or queues in Dexie offline.

4. **Hand-rolled SVG drag editor (`FlowGraphEditor`)** — no elkjs/dagre/xyflow; pure SVG + pointer events; 20px grid snap; local-state commit-on-pointer-up per Puck state-isolation pattern (RESEARCH Q5, Pitfall 5).

5. **SVG flow-graph renderer (`FlowCanvas`)** — `<svg>` with `<g transform="translate(x,y)">` per node and `<path>` per edge. Static `NODE_COLORS` map. Read-only render path. No prior SVG component in repo.

6. **Three-place contract enforcement script (`scripts/contract-check.ts`)** — parses three source files, diffs block-registration sets, fails the build on mismatch. No prior project has a contract-enforcement script; closest is the existing `src/lib/builder/puck-config.tsx:399-418` comment block that documents the invariant but does not enforce it.

7. **cmdk integration (`CommandPalette`)** — new library (`cmdk@1.1.1`); Radix-dialog-backed; three result groups with live filtering from current SOP context; global Cmd/Ctrl+K handler mounted once in worker layout.

8. **Paper/ink theme scope (`body[data-theme="paper"]`)** — cohabits with next-themes' html-level `class="dark"` (Pitfall 8); all blueprint selectors MUST scope to `body[data-theme="paper"]` unqualified, NOT `.dark body[data-theme="paper"]`. Blocks rendered inside worker routes must NOT use `dark:` Tailwind variants.
