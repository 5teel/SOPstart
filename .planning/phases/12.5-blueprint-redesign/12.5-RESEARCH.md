**Researched:** 2026-04-24
**Domain:** Next.js 16 WebSocket transport, Deepgram Nova-3 streaming, SVG flow-graph layout, MediaRecorder cross-browser, Puck 0.21 custom fields, cmdk, Supabase Storage voice buckets, RLS for new tables, next-themes coexistence

## Executive Summary

The blocking finding for planning is this: **Next.js 16 App Router still has no native WebSocket route handler support**. This is an unresolved upstream issue in `vercel/next.js`, confirmed current as of April 2026. The project cannot stream audio to Deepgram via a Next.js route handler — the connection will either 426 immediately or drop after the serverless timeout. The two viable paths are (a) a sidecar Node.js WebSocket service deployed as a separate Railway service, or (b) proxy the browser directly to Deepgram using a short-lived Deepgram token minted by a Next.js route handler. **Option (b) is the recommended path**: it eliminates the sidecar entirely, it is the exact pattern Deepgram documents for browser clients, and Railway deployment stays single-service. The Next.js side only needs a `POST /api/voice/token` endpoint that mints a 30-second scoped token via Deepgram's `/auth/grant` API; the browser then connects `wss://api.deepgram.com/v1/listen` directly.

Other resolutions: Deepgram Nova-3 **does support `language=en-NZ`** natively (BCP-47 tag confirmed in docs). Hand-roll the flow-graph layout — elkjs is 1MB+ and dagre pulls lodash; neither is worth it when D-13/D-14 already specify user-dragged 20px-grid positioning. MediaRecorder needs a `isTypeSupported`-gated fallback chain (`webm;codecs=opus` → `mp4`); Deepgram accepts both. Puck's `onChange` fires per-keystroke with no internal debounce — the `FlowGraphEditor` MUST hold its own internal state and only call `onChange` on commit boundaries (drag end, blur, button click). cmdk (pacocoursey, 1.1.1) beats kbar on bundle size and mobile semantics.

**Primary recommendation:** Skip the sidecar. Browser → Deepgram direct, token minted by Next.js. Ship 12.5 as a single Railway service; defer any WebSocket infrastructure to Phase 18 if ever needed.

## Open Research Questions — Resolved

### 1. Next.js 16 WebSocket support

**Status:** Next.js 16 App Router does NOT support WebSocket route handlers. [VERIFIED: `vercel/next.js` Discussion #58698 — the PR #58704 from Aaron Friel remains unmerged; no `upgrade()` method on `NextRequest`; route handlers run as serverless/edge functions that terminate on request completion and cannot hold a long-lived bidirectional connection.]

**Three architectural options:**

| Option | Shape | Pros | Cons |
|--------|-------|------|------|
| (a) Sidecar Node.js WebSocket service | Separate Railway service `ws.safestart.nz` running `ws` npm package; Next.js forwards requests via `fetch` with cookie header | Full control, can proxy Deepgram | 2nd Railway service, 2nd deploy, cookie domain headache, 2× cost |
| (b) **Browser → Deepgram direct via short-lived token** | `POST /api/voice/token` mints 30s Deepgram scoped token; browser connects `wss://api.deepgram.com/v1/listen` directly | Zero sidecar; matches Deepgram's documented browser pattern; Railway stays single-service | Audio bytes leave browser to Deepgram directly (not through our server), so server doesn't see raw audio stream — but D-03 uploads the blob to Supabase Storage client-side anyway |
| (c) Pusher / Ably managed relay | Browser → Ably → our Next.js webhook → Deepgram | Works on Vercel; managed | New vendor, $$$, adds latency |

**Recommendation: Option (b).** [CITED: Deepgram token-based-authentication docs — "temporary tokens are ideal for client-side/untrusted applications"; 30-second TTL makes browser exposure safe.]

**Code sketch:**

```ts
// src/app/api/voice/token/route.ts
import { NextResponse } from 'next/server'
import { createClient } from '@/lib/supabase/server'

export async function POST() {
  const supabase = await createClient()
  const { data: { user } } = await supabase.auth.getUser()
  if (!user) return NextResponse.json({ error: 'unauthorized' }, { status: 401 })

  // Mint a 30s scoped token from Deepgram
  const res = await fetch('https://api.deepgram.com/v1/auth/grant', {
    method: 'POST',
    headers: { Authorization: `Token ${process.env.DEEPGRAM_API_KEY}` },
  })
  if (!res.ok) {
    return NextResponse.json({ error: 'token_grant_failed' }, { status: 502 })
  }
  const { access_token, expires_in } = await res.json() // expires_in = 30
  return NextResponse.json({ access_token, expires_in })
}
```

```ts
// src/components/sop/VoiceCaptureControl.tsx (relevant portion)
async function startStream() {
  const { access_token } = await fetch('/api/voice/token', { method: 'POST' }).then(r => r.json())
  const params = new URLSearchParams({
    model: 'nova-3',
    language: 'en-NZ',
    encoding: 'opus',          // or 'linear16' if using PCM — see Q4
    sample_rate: '48000',
    interim_results: 'true',
    smart_format: 'true',
    punctuate: 'true',
  })
  // Browsers cannot set custom headers on WebSocket. Use Sec-WebSocket-Protocol
  // subprotocol trick per Deepgram docs.
  const ws = new WebSocket(
    `wss://api.deepgram.com/v1/listen?${params}`,
    ['token', access_token]  // Sec-WebSocket-Protocol frames; Deepgram accepts the 2nd as token
  )
  // ... wire MediaRecorder → ws.send(blob); keepalive every 3s; close on final
}
```

**Auth detail for browser WebSocket:** Browsers prohibit custom headers on `WebSocket()`. Deepgram documents the `Sec-WebSocket-Protocol: ["token", "<access_token>"]` subprotocol pattern as the browser-safe auth method. [CITED: https://developers.deepgram.com/docs/using-the-sec-websocket-protocol]

**Why NOT the sidecar:**
- The session cookie (`sb-<ref>-auth-token`) is scoped to the Next.js host; a separate Railway service on a different host cannot read it without a subdomain + shared cookie domain + CORS shenanigans.
- The sidecar would have to proxy audio to Deepgram anyway, doubling bandwidth cost.
- Railway free tier allows one service; two-service deploy triples the surface area to monitor.

**Audio blob persistence stays server-mediated:** After each capture, upload the blob to Supabase Storage via an authenticated Next.js server action (`saveVoiceNote(sopId, blob, transcript, confidence)`) — this is the path that actually needs auth + RLS. The WebSocket stream does not need our server at all.

### 2. Deepgram Nova-3 API contract

**Language: `en-NZ` IS supported natively.** [CITED: Deepgram models-languages-overview — "English: `en`, `en-US`, `en-AU`, `en-GB`, `en-IN`, `en-NZ`"]. Use `language=en-NZ` as the project default. Fallbacks through the existing `VoiceNoteBlockPropsSchema.language` enum (en-NZ / en-AU / en-US).

**WebSocket endpoint:** `wss://api.deepgram.com/v1/listen`

**Authentication:**
- REST (token grant): `Authorization: Token ${DEEPGRAM_API_KEY}`
- WebSocket (browser): `Sec-WebSocket-Protocol: ["token", "<short_lived_token>"]` — use the `WebSocket(url, protocols)` 2nd constructor arg, NOT headers.
- WebSocket (server, if we ever add sidecar): `Authorization: Bearer <token>` header.

**Query parameters (pinned for SafeStart):**

```
model=nova-3
language=en-NZ
encoding=opus              # matches MediaRecorder webm;codecs=opus output
sample_rate=48000          # MediaRecorder default
interim_results=true       # drives live "listening" UX
smart_format=true          # numbers, punctuation, capitalization
punctuate=true
no_delay=true              # reduces ~200ms latency further for UX
vad_events=true            # emits SpeechStarted / utterance_end for state machine
```

Measurement blocks additionally pass `numerals=true` to bias "twenty-two point five" → "22.5".

**Keepalive:** Send `{"type": "KeepAlive"}` as a TEXT (not binary) WebSocket frame every **3–5 seconds** during idle. Deepgram closes with `NET-0001` after **10 seconds** of silence. [CITED: Deepgram audio-keep-alive docs]

**Closing the stream:** Send `{"type": "CloseStream"}` to flush and receive final transcripts; then `ws.close()` client-side.

**Finalize (force partial-to-final without closing):** Send `{"type": "Finalize"}`. Useful for the "I'm done talking" button without tearing down the socket.

**Accepted audio input:** Both `audio/webm;codecs=opus` (MediaRecorder default on Chrome/Android) and `audio/mp4` (iOS Safari default) are accepted. [CITED: Deepgram supported-audio-formats]

**Response message shape:**

```jsonc
{
  "type": "Results",
  "channel_index": [0, 1],
  "duration": 1.23,
  "start": 0.0,
  "is_final": false,         // false = interim; true = committed
  "speech_final": false,     // true = end-of-utterance
  "channel": {
    "alternatives": [{
      "transcript": "twenty two point five",
      "confidence": 0.94,
      "words": [ /* ... per-word timing */ ]
    }]
  }
}
```

Voice state machine mapping:
- `idle` → user taps mic → `listening` (ws opens, MediaRecorder starts)
- partial results (`is_final: false`) → stay in `listening`, update rolling transcript display
- `speech_final: true` OR user taps stop → `transcribing` (send `Finalize`, wait for `is_final: true`)
- `is_final: true` received → `captured` (persist, close ws)
- error / timeout → `idle` with toast

### 3. SVG flow-graph layout library

**Recommendation: Hand-roll. Do NOT add a layout library.**

**Comparison:**

| Library | Version | Gzip | License | TS types | Verdict |
|---------|---------|------|---------|----------|---------|
| `elkjs` | 0.11.1 | ~1.4MB min / ~230KB gzip (worker-bundled Java port) | EPL-2.0 (copyleft-ish) | Shipped | TOO BIG; license friction for SaaS |
| `dagre` | 0.8.5 | ~70KB gzip (+ lodash transitive) | MIT | `@types/dagre` separate | Large; stagnant (last publish 2018) |
| `@xyflow/react` | 12.10.2 | ~150KB gzip + CSS | MIT | Shipped | Overkill — full canvas framework when we only need SVG renderer + drag |
| Hand-rolled SVG + 20px grid | — | 0 KB (already in app) | — | — | **Pick this** |

**Rationale:**
1. **D-13 already commits to 20px-grid absolute positioning** — users drag nodes themselves. Auto-layout is explicitly not required.
2. **D-15 requires `deriveFlowGraph(sop)` to return within 50ms for 500 steps.** Naive vertical stacking (y = index × 80) is O(n), trivially hits that budget.
3. The admin-side drag UX is a ~150-line React+SVG component. Puck already ships `@dnd-kit/react` transitively via `@puckeditor/core@0.21.2` — we can reuse it for drag primitives if needed [VERIFIED: Phase 12 RESEARCH.md].
4. Worker-side flow tab is pure render — SVG `<g transform="translate(x,y)">` per node, `<path d="M ... L ...">` per edge. No library needed.

**If the admin later wants auto-layout suggestions (12.6+):** revisit dagre as a lazy dynamic import (`const { default: dagre } = await import('dagre')`) so it never lands in worker bundle.

**Derivation function layout algorithm (for `deriveFlowGraph`):**

```ts
// Straight-line vertical stack with lane offsets per section.
let y = 0
for (const section of sections) {
  y += 40 // section gap
  for (const step of section.steps) {
    y += 80 // row height on 20px grid (multiple of 20)
    node.position = { x: 100, y }  // simple column
  }
}
// Edges: sequential step.i → step.i+1; decision/escalate get branch offsets later
```

[ASSUMED] The 50ms budget on 500 steps is comfortable at this complexity; no benchmark yet. Validate during Wave 5.

### 4. MediaRecorder cross-browser audio format

**Support matrix:**

| Platform | Best format | Fallback | Deepgram? |
|----------|------------|----------|-----------|
| Chrome / Edge / Chromium Android | `audio/webm;codecs=opus` | `audio/webm` | ✓ (encoding=opus) |
| Firefox | `audio/ogg;codecs=opus` | `audio/webm` | ✓ (encoding=opus, container=ogg or webm) |
| iOS Safari 14.3+ | `audio/mp4` (AAC) | — | ✓ (encoding=aac) |
| Desktop Safari 14.1+ | `audio/mp4` (AAC) | `audio/webm` in newer versions | ✓ |
| iOS Safari <14.3 | **No MediaRecorder** | — | — (graceful degrade: hide mic control) |

[CITED: WebKit blog 2020-11 MediaRecorder API — Safari supports MP4/AAC; CITED: Deepgram supported-audio-formats]

**Minimum iOS version:** iOS 14.3 (Dec 2020). Phones older than that are <1% of NZ traffic per 2026 market share; acceptable to graceful-degrade.

**Feature-detect + pick chain (pattern the planner should encode):**

```ts
// src/lib/voice/media-recorder.ts
const PREFERENCE_CHAIN = [
  { mimeType: 'audio/webm;codecs=opus', deepgramEncoding: 'opus', sampleRate: 48000 },
  { mimeType: 'audio/webm',              deepgramEncoding: 'opus', sampleRate: 48000 },
  { mimeType: 'audio/ogg;codecs=opus',   deepgramEncoding: 'opus', sampleRate: 48000 },
  { mimeType: 'audio/mp4',               deepgramEncoding: 'aac',  sampleRate: 44100 },
] as const

export function pickRecorderFormat(): typeof PREFERENCE_CHAIN[number] | null {
  if (typeof window === 'undefined' || typeof MediaRecorder === 'undefined') return null
  for (const entry of PREFERENCE_CHAIN) {
    if (MediaRecorder.isTypeSupported(entry.mimeType)) return entry
  }
  return null
}

export function isVoiceCaptureSupported(): boolean {
  return typeof navigator !== 'undefined'
    && typeof navigator.mediaDevices?.getUserMedia === 'function'
    && typeof MediaRecorder !== 'undefined'
    && pickRecorderFormat() !== null
}
```

The chosen `deepgramEncoding` and `sampleRate` feed directly into the Deepgram WebSocket query string (Q2). The stored blob's MIME goes into Supabase Storage path `{org_id}/voice/{sop_id}/{uuid}.{ext}` where `ext = webm|mp4|ogg`.

**Chunk timeslice:** Call `mediaRecorder.start(250)` for 250ms chunks — enough granularity for `ws.send(blob)` to stream, small enough to not starve Deepgram. [ASSUMED: 250ms is the widely-used default in streaming STT integrations; validate via `no_delay=true` latency measurement.]

### 5. Puck custom field state isolation

**Finding: Puck does NOT debounce or batch `onChange`.** [VERIFIED: `packages/core/components/Puck/index.tsx` on the Puck GitHub — the effect subscribes to `appStore.state.data` changes and calls `onChange(data)` synchronously after a `deepEqual` check.] Each keystroke / drag / field edit in any Puck field fires onChange immediately.

**Implication for FlowGraphField:** if we naively call `puckOnChange(newGraph)` on every mouse-move during a node drag, we (a) re-serialize the entire graph every frame, (b) trigger Dexie writes through `useBuilderAutosave` (already 750ms debounced — fine), (c) trigger Puck's deepEqual scan on a potentially-large graph object, and (d) may cause visible stutter on low-end admin machines.

**Recommendation: FlowGraphEditor owns its own state machine.** Puck gives us a `value` and `onChange`; we treat `value` as the "committed" graph only, hold the "live" graph in local React state, and call `onChange` only at commit boundaries:

```tsx
// src/lib/builder/flow-graph-field.tsx
'use client'
import { useState, useEffect, useRef } from 'react'
import type { FieldLabelProps } from '@puckeditor/core'

export function FlowGraphField({ value, onChange }: {
  value: FlowGraph | undefined
  onChange: (next: FlowGraph) => void
}) {
  const [liveGraph, setLiveGraph] = useState<FlowGraph>(() => value ?? emptyGraph())
  const committedRef = useRef(value)

  // Sync down: if Puck pushes a new value (e.g. undo), reset local state
  useEffect(() => {
    if (value && !deepEqual(value, committedRef.current)) {
      setLiveGraph(value)
      committedRef.current = value
    }
  }, [value])

  // Commit boundaries only:
  //   - node drag END (onMouseUp / onPointerUp), not onMouseMove
  //   - edge connection completed
  //   - label input blur (not onChange)
  //   - explicit "Save" button
  const commit = (next: FlowGraph) => {
    setLiveGraph(next)        // update view
    committedRef.current = next
    onChange(next)            // tell Puck — single batched call
  }

  return <FlowGraphCanvas graph={liveGraph} onDragEnd={commit} onEdgeCreate={commit} ... />
}
```

**Key patterns:**

1. **Drag preview is local-only.** While dragging, update `liveGraph` via `setLiveGraph`; do NOT call `onChange`. Only commit on pointer up.
2. **Text inputs use uncontrolled pattern or onBlur commit.** `<input defaultValue={node.label} onBlur={(e) => commit(updateNodeLabel(liveGraph, node.id, e.target.value))} />`.
3. **Every commit is a Puck undo step.** That's fine — admins expect "moved node A from (100,200) to (100,400)" as one undo, not 847.
4. **A 200ms `requestAnimationFrame`-scoped debounce on commit is optional polish** to coalesce rapid edits (e.g. a label typed fast and then blurred) — likely unnecessary, but plan for it if the admin fleet reports sluggishness.

[CITED: Puck custom-fields docs — `onChange(value, ui)` signature confirmed; no documented debounce or batching from Puck.]

**Note on Puck's deepEqual guard:** Puck does call `deepEqual(data, previousData.current)` before firing onChange, so identical consecutive values don't re-emit — but on a 500-node graph, deepEqual itself is O(n), so calling onChange on every mousemove is still expensive.

### Server action shape for `updateSopFlowGraph`

**Match Phase 12's `updateSectionLayout` pattern verbatim.** [VERIFIED: `src/actions/sections.ts:147-207`]

```ts
// src/actions/sops/flow-graph.ts
'use server'
import { z } from 'zod'
import { createClient } from '@/lib/supabase/server'
import { FlowGraphSchema } from '@/lib/validators/flow-graph'

const Input = z.object({
  sopId: z.string().uuid(),
  flowGraph: z.union([FlowGraphSchema, z.null()]),  // allow clearing
  clientUpdatedAt: z.number().int().min(0),
})

const MAX_GRAPH_BYTES = 256 * 1024 // 256 KB — slightly bigger than layout cap; graphs can have many edges

export async function updateSopFlowGraph(input: z.infer<typeof Input>) {
  const parsed = Input.safeParse(input)
  if (!parsed.success) return { error: parsed.error.issues[0]?.message ?? 'Invalid input' }

  const serialized = JSON.stringify(parsed.data.flowGraph)
  if (Buffer.byteLength(serialized, 'utf8') > MAX_GRAPH_BYTES) {
    return { error: 'Flow graph exceeds 256 KB; reduce node/edge count' }
  }

  const supabase = await createClient()
  const { data: { user } } = await supabase.auth.getUser()
  if (!user) return { error: 'Not authenticated' }

  // Role gate — same JWT claim pattern as updateSectionLayout
  const { data: { session } } = await supabase.auth.getSession()
  const claims = session?.access_token ? JSON.parse(atob(session.access_token.split('.')[1])) : {}
  if (!['admin', 'safety_manager'].includes(claims['user_role'])) {
    return { error: 'Admin access required' }
  }

  // LWW check (D-07 precedent)
  const { data: current } = await supabase
    .from('sops')
    .select('updated_at')
    .eq('id', parsed.data.sopId)
    .maybeSingle()
  if (current?.updated_at) {
    const serverMs = new Date(current.updated_at as string).getTime()
    if (serverMs > parsed.data.clientUpdatedAt) return { error: 'server_newer' }
  }

  const { error } = await supabase
    .from('sops')
    .update({ flow_graph: parsed.data.flowGraph as unknown, updated_at: new Date(parsed.data.clientUpdatedAt).toISOString() })
    .eq('id', parsed.data.sopId)
  if (error) return { error: error.message }
  return { success: true as const }
}
```

### Offline voice queue — Deepgram REST endpoint

**Offline flush uses `POST https://api.deepgram.com/v1/listen` with the blob body** (not WebSocket). Same query params (`model=nova-3&language=en-NZ&smart_format=true&punctuate=true`).
```ts
export async function flushVoiceNoteQueue(supabase: AnySupabaseClient) {
  const dirty = await db.voiceNotesQueue.where('syncState').equals('dirty').toArray()
  for (const note of dirty) {
    try {
      // 1. Mint token
      const { access_token } = await fetch('/api/voice/token', { method: 'POST' }).then(r => r.json())
      // 2. POST blob to Deepgram REST
      const params = new URLSearchParams({ model: 'nova-3', language: 'en-NZ', smart_format: 'true', punctuate: 'true' })
      const res = await fetch(`https://api.deepgram.com/v1/listen?${params}`, {
        method: 'POST',
        headers: { Authorization: `Bearer ${access_token}`, 'Content-Type': note.audioMime },
        body: note.blob,
      })
      const json = await res.json()
      const transcript = json.results?.channels[0]?.alternatives[0]?.transcript ?? ''
      const confidence = json.results?.channels[0]?.alternatives[0]?.confidence
      // 3. Upload blob + insert row via saveVoiceNote server action
      await saveVoiceNote({ ...note, transcript, confidence })
      await db.voiceNotesQueue.update(note.id, { syncState: 'synced' })
    } catch (err) { /* leave dirty, retry next flush */ }
  }
}
```

## Risks & Pitfalls

### Pitfall 1: Using Next.js route handler for WebSocket
**What goes wrong:** Planner writes `src/app/api/voice/ws/[sopId]/route.ts` that `upgrade()`s the request. Deploy to Railway, connection 426s immediately.
**Why:** Next.js 16 App Router has no WebSocket primitive. `NextRequest` has no `upgrade()` method.
**How to avoid:** Browser → Deepgram direct via scoped token. The only Next.js endpoint needed is `POST /api/voice/token` (plain request/response).

### Pitfall 2: Exposing `DEEPGRAM_API_KEY` to the browser
**What goes wrong:** Variable named `NEXT_PUBLIC_DEEPGRAM_API_KEY` ends up in client bundle; key leaks; $$$.
**Why:** Typo / misunderstanding of Next.js env rules.
**How to avoid:** Use `DEEPGRAM_API_KEY` (no `NEXT_PUBLIC_` prefix). CI grep: `grep -r "NEXT_PUBLIC_DEEPGRAM" src/` must return 0.

### Pitfall 3: Custom Authorization header on WebSocket from browser
**What goes wrong:** Code tries `new WebSocket(url, { headers: {...} })` — second arg is protocols, not options; headers silently ignored; Deepgram 401s.
**Why:** Browsers don't allow custom headers on WS; this is an HTML5 spec limitation.
**How to avoid:** Use `new WebSocket(url, ['token', access_token])` — Deepgram's `Sec-WebSocket-Protocol` trick. Or pass token in query string (less preferred — logged in server access logs).

### Pitfall 4: Forgetting Deepgram keepalive → 10s NET-0001 disconnect
**What goes wrong:** User taps mic, says nothing for 12s, connection drops, recording lost.
**Why:** Deepgram closes idle streams after 10 seconds.
**How to avoid:** `setInterval(() => ws.send(JSON.stringify({type:'KeepAlive'})), 3000)` while state is `listening`. Clear on close.

### Pitfall 5: Puck `onChange` on every mouse-move in FlowGraphEditor
**What goes wrong:** Dragging a node fires onChange 60 times/second; Puck's `deepEqual` runs on a 500-node graph per frame; admin browser locks up.
**Why:** Puck does not batch or debounce onChange.
**How to avoid:** FlowGraphEditor owns local state; only calls `onChange` on pointer-up / edge-complete / blur / explicit save. See Q5 pattern.

### Pitfall 6: iOS Safari MediaRecorder < 14.3 crashes
**What goes wrong:** `new MediaRecorder(stream)` throws `ReferenceError` on iOS 14.2 and below; voice control silently breaks.
**Why:** MediaRecorder landed in iOS 14.3 (Dec 2020).
**How to avoid:** `isVoiceCaptureSupported()` feature-detect at render time; hide mic button + show `Voice capture requires iOS 14.3+` hint on unsupported devices.

### Pitfall 7: `<input>` inside FlowGraphField calls onChange on every keystroke
**What goes wrong:** Admin types a 40-char node label; 40 onChange calls; 40 Dexie writes; 40 Puck history entries.
**Why:** React controlled inputs fire onChange per keystroke unless wrapped.
**How to avoid:** Uncontrolled (`defaultValue`) + commit on `onBlur`, OR controlled locally + commit on `onBlur`.

### Pitfall 8: Mixing next-themes `class="dark"` with manual `data-theme`
**What goes wrong:** Worker route loads with both `class="dark"` on html and `data-theme="paper"` on body; Tailwind dark: variants inside blueprint blocks render as dark even though the body wants paper.
**Why:** next-themes keeps `class="dark"` regardless of nested layouts.
**How to avoid:** In `body[data-theme="paper"]` CSS, explicitly neutralize `--tw-` variables. OR more pragmatic: `<html class="dark" data-theme="paper">` is fine as long as blueprint-theme.css selectors are authored against `body[data-theme="paper"]` and NEVER against `.dark body[data-theme="paper"]`. Tailwind dark: utility classes inside sop block components should be absent — those blocks are paper-only.

### Pitfall 9: Voice blob MIME ↔ Deepgram encoding mismatch
**What goes wrong:** MediaRecorder falls back to `audio/mp4` on iOS but client code always sends `encoding=opus` in the WS query string; Deepgram returns empty transcripts.
**Why:** Encoding is container-specific.
**How to avoid:** Pass the recorder format (from `pickRecorderFormat()`) into the WebSocket URL builder; use the matching `deepgramEncoding` field. See Q4 pattern.

### Pitfall 10: Derivation runtime on pathological SOPs
**What goes wrong:** A SOP with 8 sections × 60 steps × 15 blocks each = 7200 items; `deriveFlowGraph` allocates a flat array + map; 180ms later the UI finally renders.
**Why:** Naive O(n·m) walking of `layout_data.content[]` per step.
**How to avoid:** Single pass over `sop.sections → steps`; skip any drill into `layout_data` for derivation unless block-type differentiation actually needs it. Memoize via TanStack Query (D-15). Perf test at 500 steps in Wave 0.

### Pitfall 11: Explicit `flow_graph` JSONB drift after schema evolution
**What goes wrong:** Phase 13 adds a new node type (e.g. `tool`); existing `sops.flow_graph` rows with `version: 1` only know 7 types; `safeParse` fails; warn-once fires; worker falls back to derived (correct behavior per D-16, but admins may be confused).
**Why:** JSONB is versioned at the `version: 1` literal.
**How to avoid:** D-16 already handles this (fallback + warn). When Phase 13 ships new node types, bump `FlowGraphSchema.version` to 2 and add a migrate() helper. Document in Phase 13 research.

## Code Examples

### Deepgram WebSocket client (browser)

```ts
// src/lib/voice/deepgram-stream.ts
'use client'
import { pickRecorderFormat } from './media-recorder'

export interface StreamHandle {
  stop: () => Promise<{ transcript: string; confidence: number; blob: Blob; ext: 'webm' | 'mp4' | 'ogg' }>
  onPartial: (cb: (text: string) => void) => void
  onFinal: (cb: (text: string, confidence: number) => void) => void
  onError: (cb: (err: Error) => void) => void
}

export async function startVoiceStream(opts: { language: 'en-NZ' | 'en-AU' | 'en-US' }): Promise<StreamHandle> {
  const format = pickRecorderFormat()
  if (!format) throw new Error('Voice capture not supported')

  // 1. Mint 30s token
  const tokenRes = await fetch('/api/voice/token', { method: 'POST' })
  if (!tokenRes.ok) throw new Error('token_grant_failed')
  const { access_token } = await tokenRes.json()

  // 2. Open ws with Sec-WebSocket-Protocol trick
  const params = new URLSearchParams({
    model: 'nova-3',
    language: opts.language,
    encoding: format.deepgramEncoding,
    sample_rate: String(format.sampleRate),
    interim_results: 'true',
    smart_format: 'true',
    punctuate: 'true',
    no_delay: 'true',
    vad_events: 'true',
  })
  const ws = new WebSocket(`wss://api.deepgram.com/v1/listen?${params}`, ['token', access_token])

  // 3. Keepalive every 3s
  const keepaliveTimer = setInterval(() => {
    if (ws.readyState === WebSocket.OPEN) ws.send(JSON.stringify({ type: 'KeepAlive' }))
  }, 3000)

  // 4. getUserMedia + MediaRecorder
  const stream = await navigator.mediaDevices.getUserMedia({ audio: true })
  const recorder = new MediaRecorder(stream, { mimeType: format.mimeType })
  const chunks: Blob[] = []

  recorder.addEventListener('dataavailable', (ev) => {
    if (ev.data.size > 0) {
      chunks.push(ev.data)
      if (ws.readyState === WebSocket.OPEN) ws.send(ev.data)
    }
  })
  recorder.start(250)  // 250ms chunks

  // ... wire ws.onmessage → partial/final callbacks; return handle
  return { /* ... */ } as StreamHandle
}
```
