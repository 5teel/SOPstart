## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] contract-check.ts parsing required 3 iterations**
- **Found during:** Task 3
- **Issue 1:** Initial regex `BLOCK_REGISTRY[^=]*=\s*\{` failed because the type annotation `Record<string, {...}>` contains `>` which isn't matched by `[^=]*`. Fixed to `BLOCK_REGISTRY[^=]+=\s*\{`.
- **Issue 2:** `extractTopLevelPascalKeys` used 2-space indent regex but puck-config.tsx uses 4-space indent. Fixed to line-by-line parsing that auto-detects indent level.
- **Issue 3:** `extractValidatorKinds` scanned all `z.literal()` in the file, so it picked up kinds even when the schema wasn't in the discriminatedUnion array. Fixed to parse only the `discriminatedUnion([...])` array body, map schema names to their `kind` literals, which correctly detects when a schema exists but isn't registered in the union.
- **Files modified:** scripts/contract-check.ts

**2. [Rule 2 - Adjustment] Legacy Phase 12 blocks excluded from validator check**
- **Found during:** Task 3
- **Issue:** TextBlock, HeadingBlock, PhotoBlock, CalloutBlock are in puckConfig.components and BLOCK_REGISTRY but NOT in BlockContentSchema (they predate the discriminated union and are layout-only). The contract-check would always report them as broken.
- **Fix:** Added to `EXCLUDED_FROM_VALIDATORS` set with a clear comment explaining they are Phase 12 legacy layout-only blocks. New Phase 12.5+ blocks still MUST have kind entries.
- **Files modified:** scripts/contract-check.ts

## Wave 4 Wiring Notes

- **MeasurementBlock:** `data-voice-target="measurement"` + `data-wave4-wiring="VoiceCaptureControl"` on the mic placeholder button. Wave 4 queries `[data-voice-target="measurement"]` and replaces with `<VoiceCaptureControl>`.
- **VoiceNoteBlock:** `data-voice-target="note"` + `data-wave4-wiring="VoiceCaptureControl"` on the Record button. Same Wave 4 wiring pattern.
- **EscalateBlock:** `data-escalation-mode={escalationMode}` + `data-wave4-wiring="escalation-dispatch"`. Wave 4 reads the mode and dispatches the appropriate server action (alert/lock/form).

## Known Stubs

- All block rendering uses `color-mix(in srgb, var(--accent-*) N%, white)` for backgrounds. These CSS vars (`--accent-measure`, `--accent-decision`, etc.) are defined in `blueprint-theme.css` (Wave 1). If a block renders outside `body[data-theme="paper"]`, the accent vars fall back to browser defaults (typically transparent). This is expected behavior — all worker routes have the paper theme mount.
- `tests/sb-ux-blocks.test.ts` and `tests/sb-ux-contract.test.ts` are structural tests. The `/api/schema` tests require a running dev server and are wired to the `phase12.5-stubs` Playwright project (requires `baseURL` config). The contract test (`sb-ux-contract.test.ts`) runs entirely via `execSync` and does not need a server.
