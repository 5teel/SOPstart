## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Reskin pages but not their child components | Reskinning a page (e.g. /activity) while its shared components (CompletionHistoryCard, ActivityFilter) still use steel-* means the page looks broken even though the page file is correct | advisory | Before declaring a page reskinned, grep its rendered component tree for steel-* / brand-yellow / neuro-panel |
| bg-steel-900 on outer layout fighting paper theme | The (protected)/layout.tsx outer div had bg-steel-900 which overrode body[data-theme=paper] background | advisory | Use layout-shell CSS class + blueprint-theme.css override. Admin routes keep dark bg; worker routes get paper override. |

## Critical Anti-Patterns (do NOT repeat these)
- **RESKIN PAGE NOT COMPONENTS**: When reskinning a page, always audit the full rendered component tree — grep for `steel-` and `brand-yellow` in every component the page imports.
- **LAYOUT BG FIGHT**: Never put `bg-steel-900` directly on a layout div that wraps both admin and worker routes. Use the layout-shell CSS class pattern.
