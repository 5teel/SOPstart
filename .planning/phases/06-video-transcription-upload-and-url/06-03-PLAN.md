**5. YouTube submit handler:**

```typescript
async function handleYoutubeSubmit() {
  setYoutubeError(null)

  if (!youtubeUrl.trim()) {
    setYoutubeError('Please enter a YouTube URL')
    return
  }
  if (!termsChecked) {
    setYoutubeError('Please confirm rights before proceeding.')
    return
  }

  // Client-side URL validation
  try {
    const u = new URL(youtubeUrl)
    const validHosts = ['www.youtube.com', 'youtube.com', 'youtu.be', 'm.youtube.com']
    if (!validHosts.includes(u.hostname)) {
      setYoutubeError('Only YouTube URLs are supported. Upload the video file directly, or paste a YouTube link.')
      return
    }
  } catch {
    setYoutubeError("That doesn't look like a YouTube URL. Check the link and try again.")
    return
  }

  setYoutubeFetching(true)
  try {
    const res = await fetch('/api/sops/youtube', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        url: youtubeUrl,
        organisationId: /* from context or prop */,
        uploadedBy: /* from context or prop */,
      }),
    })
    const data = await res.json()

    if (data.noCaption) {
      setYoutubeError(data.message)
      return
    }

    if (data.error) {
      setYoutubeError(data.error)
      return
    }

    if (data.sopId) {
      // Navigate to review page
      window.location.href = `/admin/sops/${data.sopId}/review`
    }
  } catch {
    setYoutubeError('Network error — please try again.')
  } finally {
    setYoutubeFetching(false)
  }
}
```

Note: The organisationId and uploadedBy values need to come from props or context. Check how the existing UploadDropzone obtains these — it may already have them from the createUploadSession server action. If not, add props to UploadDropzone or fetch from the user session.

**8. Video upload flow (when video file is queued):**

For video files, the upload flow differs:
1. Call `extractAudioFromVideo(file)` from `@/lib/parsers/extract-video-audio` to extract audio client-side
2. Call `createVideoUploadSession` server action to get sopId and storage path
3. Use existing `tusUpload` from `@/lib/upload/tus-upload` to upload the audio MP3 to `sop-videos` bucket
4. On TUS completion, call `fetch('/api/sops/transcribe', { body: { sopId } })` to trigger the transcription pipeline

This is different from the existing document upload flow which uses presigned URLs. Video always uses TUS because the extracted audio can still be large.
