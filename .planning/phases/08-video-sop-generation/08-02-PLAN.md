**src/lib/video-gen/render-slides.ts** — Narrated slideshow Shotstack timeline builder (VGEN-01, D-02):

Export `buildSlideshowEdit(sections: SectionWithAudio[]): ShotstackEdit`.

Build two tracks:
- Track 1 (video): one HTML clip per section. Each clip is an HTML document sized 1280x720 with:
  - Background: `#111827` (steel-900)
  - Title: `#f59e0b` (brand-yellow), 36px, font-family sans-serif
  - Content: `#f3f4f6` (steel-100), 22px, line-height 1.6
  - Section content rendered as the `contentHtml` field (already formatted)
  - Padding: 48px
  - `start`: cumulative duration of all prior sections
  - `length`: this section's `audioDuration`
- Track 2 (audio): one audio clip per section with matching start/length, asset type `'audio'` pointing to `audioStorageUrl`

Return the complete `ShotstackEdit` object with `output: { format: 'mp4', resolution: 'hd' }`.

**src/lib/video-gen/render-scroll.ts** — Screen-recording-style timeline builder (VGEN-02, D-03):

Export `buildScrollEdit(sections: SectionWithAudio[], totalDuration: number, stitchedAudioUrl: string): ShotstackEdit`.

Build a single-track timeline:
- Track 1 (video): single HTML clip covering the full SOP content. The HTML renders all sections vertically with CSS keyframe animation for scrolling. Use a `@keyframes scroll` animation that translates Y position proportionally to each section's time boundary:
  - Section N starts at cumulative time T, so at `T/totalDuration * 100`% of the animation the content scrolls to section N's position
  - Same visual styling as slideshow (steel-900 bg, brand-yellow headings, steel-100 text)
  - Width 1280px, content height auto (scrolls within the frame)
  - Clip starts at 0, length is `totalDuration`
- Soundtrack: `{ src: stitchedAudioUrl, effect: 'fadeIn' }` — the stitched full-SOP audio file

Return the complete `ShotstackEdit` object.

**src/lib/video-gen/pipeline.ts** — Orchestration pipeline:

Export `runVideoGenerationPipeline(jobId: string): Promise<void>`.

Pipeline steps:
1. **Analyzing** — Update job status to `analyzing`. Fetch the SOP and its sections from Supabase (admin client). Sort sections by sort_order. Build the text script per section: for 'steps' sections, format each step as a numbered sub-bullet. Compute `contentHtml` (for slides) and `contentText` (for TTS) per section.

2. **Generating Audio** — Update job status to `generating_audio`, current_stage to `'generating_audio'`. Run TTS for ALL sections in parallel via `Promise.all(sections.map(s => generateSectionAudio(s.contentText, s.sectionType)))`. For each result, upload the audio buffer to Supabase Storage at `{orgId}/{sopId}/audio/{sectionId}.mp3` in the `sop-generated-videos` bucket. Build `SectionWithAudio[]` array with storage URLs and durations.

3. **Compute chapters** (VGEN-04): Build `ChapterMarker[]` from cumulative section durations:
   ```typescript
   let cumulative = 0
   const chapters = sectionsWithAudio.map(s => {
     const marker = { sectionId: s.sectionId, title: s.title, timestamp: cumulative }
     cumulative += s.audioDuration
     return marker
   })
   ```
   Store chapters in `video_generation_jobs.chapter_markers` JSONB column.

4. **Building timeline** — Based on job format:
   - `narrated_slideshow`: call `buildSlideshowEdit(sectionsWithAudio)`
   - `screen_recording`: concatenate all audio buffers into a single MP3 file (binary append of Buffer objects), upload as `{orgId}/{sopId}/audio/stitched.mp3`, then call `buildScrollEdit(sectionsWithAudio, totalDuration, stitchedAudioUrl)`

5. **Rendering** — Update job status to `rendering`, current_stage to `'rendering'`. Call `submitShotstackRender(edit)` to get the Shotstack render ID. Store render ID in `video_generation_jobs.shotstack_render_id`.

6. **Polling** — Poll `getShotstackRender(renderId)` every 5 seconds with a 240-second timeout guard. When status is `'done'`:
   - Download the rendered video from Shotstack URL
   - Re-upload to Supabase Storage at `{orgId}/{sopId}/video/{jobId}.mp4` in `sop-generated-videos` bucket
   - Update job: `status = 'ready'`, `video_url = storageUrl`, `completed_at = now()`, `current_stage = 'ready'`
   When status is `'failed'`: Update job: `status = 'failed'`, `error_message = error`, `current_stage = 'failed'`

7. **Error handling** — Wrap entire pipeline in try/catch. On any error, update job to `status = 'failed'` with the error message. Use the admin Supabase client for all DB writes (bypasses RLS for status updates).

All `video_generation_jobs` status updates use `updated_at: new Date().toISOString()` to trigger Supabase Realtime events for the client UI.

**src/app/api/sops/generate-video/route.ts** — POST handler (VGEN-06):

Follow the same pattern as `/api/sops/parse/route.ts` (fire-and-forget async pipeline).

1. Parse and validate request body with `generateVideoSchema`
2. Auth check: get user from Supabase server client, verify admin/safety_manager role via JWT app_metadata
3. Fetch the SOP to confirm it exists and is published (`status = 'published'`)
4. **Idempotency check (D-14):** Query `video_generation_jobs` for existing job with same `(sop_id, format, sop_version)` where status is NOT 'failed'. If found, return 200 with the existing job ID.
5. Create new `video_generation_jobs` record with admin client:
   - status: 'queued'
   - sop_version: sop.version (current version number)
   - created_by: user.id
   - organisation_id: from user's JWT
6. Fire-and-forget: Call `runVideoGenerationPipeline(jobId)` without awaiting (the pipeline updates its own status via DB writes). Use `void runVideoGenerationPipeline(jobId)` pattern with `.catch()` to prevent unhandled rejection.
7. Return 202 with `{ jobId: newJob.id }`

**src/actions/video.ts** — Server actions for video operations:

`'use server'` directive at top.

Export `recordVideoView(input: { sopId: string, sopVersion: number, videoJobId: string })`:
- Validate input with `recordVideoViewSchema`
- Get authenticated user from server Supabase client
- Insert into `sop_completions`:
  - `id`: crypto.randomUUID() (client-generated UUID for idempotent retry)
  - `sop_id`: input.sopId
  - `user_id`: user.id
  - `organisation_id`: from JWT
  - `sop_version`: input.sopVersion
  - `completion_type`: 'video_view'
  - `video_job_id`: input.videoJobId
  - `status`: 'pending_sign_off' (matches existing column constraint)
  - `completed_at`: new Date().toISOString()
- On 23505 conflict (duplicate): silently succeed (idempotent)
- On any other error: silently fail (fire-and-forget per D-15 — do not throw to client)

Export `publishVideo(jobId: string)`:
- Auth check: admin/safety_manager
- Update `video_generation_jobs` SET `published = true` WHERE `id = jobId` and status = 'ready'
- Return `{ success: true }`

Export `unpublishVideo(jobId: string)`:
- Auth check: admin/safety_manager
- Update `video_generation_jobs` SET `published = false` WHERE `id = jobId`
- Return `{ success: true }`

Export `deleteVideoJob(jobId: string)`:
- Auth check: admin/safety_manager
- Delete the job record and associated Storage files (audio + video) using admin client
- Return `{ success: true }`

Export `regenerateVideo(sopId: string, format: VideoFormat)`:
- Auth check: admin/safety_manager
- Mark existing job(s) for this SOP+format as failed (to clear the unique constraint)
- Call the generate-video API route internally or create a new job directly
- Return `{ jobId: newJob.id }`
