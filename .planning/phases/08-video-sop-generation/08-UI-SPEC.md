## Spacing Scale

**Scale:** 8-point base. All spacing in multiples of 4px. Carried forward unchanged from Phases 02–07.

|-------|-------|----------------|-------|
| xs | 4px | `gap-1` / `p-1` | Icon gaps, inline padding, chapter timestamp gap |
| sm | 8px | `gap-2` / `p-2` | Compact element spacing, stage step gaps |
| md | 12px | `p-3` | Component padding (tight) — prior-phase pattern |
| md+ | 16px | `p-4` | Component padding (standard), card inner padding |

| 2xl | 32px | `p-8` | Generous container padding |

| Value | Tailwind | Rationale |

Carried forward from Phase 07 — no new tokens added.

**60% dominant surface:** `steel-900` (`#111827`) — page background, video player letterbox, generation job list background

## Interaction Contract

### Admin — Video Generation Flow

| Trigger | Response |
|---------|----------|
| Admin taps "Video" from SOP library row (published SOP) | Navigate to `/admin/sops/{sopId}/video` |
| Admin arrives at video page — no video generated | Show format selector + disabled "Generate video" CTA |
| Admin selects a format card | Card border turns `brand-yellow`; radio fills; "Generate video" CTA enables |
| Admin taps "Generate video" | Disable CTA; show `VideoGenerationStatus` component with stage stepper at step 1 (Analysing); call `/api/sops/generate-video` with `{ sopId, format }` |
| Generation in progress | Stage stepper advances via Supabase Realtime on `video_generation_jobs` table updates (same pattern as `parse_jobs`). Polling fallback after 5s if Realtime silent. |
| Generation completes | Stage stepper all green; show `VideoAdminPreview` with video player and action bar |
| Generation fails | Failure card: stage name of failure + brief error + "Re-generate" link + "Delete" link |
| Admin taps "Re-generate" (after failure or from preview) | Inline confirm: "Re-generate this video? The current video will be replaced." — [Yes, re-generate] (`bg-brand-orange`) + [Keep current video] (`bg-steel-700`) |
| Admin confirms re-generate | Create new generation job (idempotency D-14: server checks for in-progress job first); show stage stepper from step 1 |
| Admin taps "Publish video" | Inline confirm: "Publish this video? Workers will see it in the SOP video tab." — [Yes, publish] (`bg-brand-yellow text-steel-900`) + [Not yet] (`bg-steel-700`) |
| Admin confirms publish | Update `video_generation_jobs.published = true`; navigate back to `/admin/sops` with success toast: "Video published." |
| Admin arrives at video page — video already generated and published | Show `VideoAdminPreview` with existing video (no format selector); show "Re-generate" and "Unpublish" buttons |
| Admin arrives at video page — video already generated but not published | Show `VideoAdminPreview` with existing video; show "Re-generate" and "Publish video" buttons |
| "Video is outdated" condition (SOP updated_at > video generated_at — D-10) | Show `VideoOutdatedBanner` above preview; badge on "Video" tab in library row action area; `brand-orange` dot on Video tab in worker SOP view |

### Worker — Video Viewing Flow

| Trigger | Response |
|---------|----------|
| Worker opens SOP detail page | If published video exists: show "Video" tab in `SopSectionTabs`. If no published video: no Video tab. |
| Worker taps "Video" tab | Render `VideoTabPanel` with player and chapter list |
| Worker taps a chapter row | Seek video to chapter start timestamp; apply active chapter highlight (yellow left border) |
| Video plays past a chapter start | Auto-apply active highlight to current chapter; scroll chapter into view if not visible |
| Worker changes playback speed | Set `video.playbackRate`; update active speed button to `bg-brand-yellow text-steel-900` |
| Worker plays video past 80% | Record `video_view` completion event in `sop_completions` (D-15) — fire-and-forget server action |
| "Video is outdated" condition | Show `VideoOutdatedBanner` above video player in worker view — informational only, no action for workers |

### Destructive Actions

| Action | Approach |
|--------|----------|
| Re-generate (replaces existing video) | Inline confirm within `VideoAdminPreview`: "Re-generate this video? The current video will be replaced." — [Yes, re-generate] (`bg-brand-orange`) + [Keep current video] (`bg-steel-700`). No modal dialog. |
| Delete generated video | Inline confirm: "Delete this generated video? Workers will no longer see a video for this SOP." — [Yes, delete] (`bg-red-600 text-steel-100`) + [Keep video] (`bg-steel-700`). Only available from failed state or admin video page (not from review page). |
| Publish video | Inline confirm: "Publish this video? Workers will see it in the SOP video tab." — [Yes, publish] (`bg-brand-yellow text-steel-900`) + [Not yet] (`bg-steel-700`). |

## Completion Tracking Contract (D-15)

**Trigger:** Worker video playback reaches 80% of total duration.

**Implementation:**
- Listen to `<video>` `timeupdate` event
- When `video.currentTime / video.duration >= 0.8` and not yet fired: call `recordVideoCompletion` server action
- Fire once per session per video (track with local ref: `hasRecordedCompletion`)
- Server action creates `sop_completions` record with:
  - `completion_type: 'video_view'`
  - `video_generation_job_id`: from props
  - `sop_version_id`: from props
  - Same append-only RLS as all completions (COMP-07)
- Server action failure: silent — do not surface error to worker (fire-and-forget)

## Accessibility Contract

- `VideoTabPanel` `<video>`: `aria-label="SOP procedure video"` on the element
- Playback speed buttons: `aria-label="{speed} playback speed"` on each; `aria-pressed={isActive}` on active speed
- Chapter list container: `role="list"` on the `<div>`; `role="listitem"` wrapper on each chapter row
- Chapter button: `aria-label="{section name} — jump to {M:SS}"` on each `<button>` element; `aria-current="true"` on active chapter
- `VideoOutdatedBanner`: `role="status"` on the banner container; screen reader will announce on render
- `VideoGenerationStatus` stepper: each step has `aria-current="step"` on active step, `aria-label="{stage name}"` on each step element
- `VideoGeneratePanel` format cards: use `<label>` wrapping `<input type="radio">` for native keyboard + screen reader support; `<fieldset>` + `<legend>` for the card group
- Inline confirm panels: focus trap within confirm area; first button receives focus on appear; `role="alertdialog"` `aria-modal="true"` on the confirm container
- "Publish video" and "Re-generate" buttons during loading: `aria-busy="true"` while request in flight
- "Video is outdated" dot on Video tab: `aria-label="Video is outdated"` — do not use title attribute alone for accessibility (supplement with visible label or tooltip that is keyboard-focusable)
