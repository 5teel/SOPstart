## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), badge padding (`px-2 py-0.5`) |
| sm | 8px | Compact row padding, icon-to-text gaps (`gap-2`) |
| md | 16px | Card padding (`p-4`), default horizontal layout padding |
| lg | 24px | Section padding (`py-6`), between version rows |
| xl | 32px | Page vertical padding (`py-8`), between major sections |
| 2xl | 48px | Major section breaks (empty states, page intros) |
| 3xl | 64px | Not used in this phase (admin panel stays compact) |

Exceptions:
- Confirm button height: 44px (`h-[44px]`) — inline confirmation dialogs (matches existing `VideoAdminPreview` pattern)
- Sticky header height: 56px (`h-[56px]`) — matches existing admin page header
- Touch targets for primary CTAs: 72px (`h-[72px]`) — matches existing `VideoGeneratePanel` and `VideoAdminPreview` glove-friendly buttons

Source: Codebase scan of `VideoGeneratePanel.tsx`, `VideoAdminPreview.tsx`, `StatusBadge.tsx`. All panel text uses `text-sm` (14px) body and `text-xs` (12px) for labels/badges. The `text-base` (16px) + `font-semibold` size is reserved for primary CTA buttons only.

Display headings (`text-lg` / 18px) are not used in this phase — the version list is a compact admin management surface.

## Component Inventory

New components introduced in this phase:

### VideoVersionList
- Container for all non-archived version rows
- Renders rows in descending `version_number` order (newest first)
- Accepts `versions: VideoGenerationJob[]` and `archivedVersions: VideoGenerationJob[]`
- Includes collapsible archived section at the bottom

### VideoVersionRow
Single version row. Compact horizontal layout:

```
[v1] [Format badge] [Status badge] [Label or "Add label..."] [Date] [Actions]
```

Layout rules:
- Fixed height: 52px for ready/failed rows; auto-height when inline confirm or label editor is open
- Version number: `text-xs font-semibold text-steel-400` — "v1", "v2", "v3"
- Format badge: `text-xs px-2 py-0.5 rounded bg-steel-700 text-steel-400` — "Narrated slideshow" | "Screen recording"
- Status badge: reuse existing `StatusBadge` variant map (add `generating` → `bg-blue-500/20 text-blue-400 animate-pulse`)
- Label text: `text-sm text-steel-100` when set; `text-sm text-steel-600 italic` when empty ("Add label...")
- Created date: `text-xs text-steel-400` — format "7 Apr 2026"
- Actions: icon buttons (Lucide), 32px tap target each — see Interaction Contracts below

Published version row differentiation:
- Left border: `border-l-2 border-brand-yellow`
- Background: `bg-steel-800` (same as other rows but visually separated by the left border)
- No separate "published" label beyond the status badge

Active/generating version row:
- Renders `VideoGenerationStatus` stepper inline below the row (expand on click or always visible)
- Disable archive action on active rows (tooltip: "Wait for generation to complete")

### ArchivedVersionsSection
Collapsible section at the bottom of `VideoVersionList`:
- Trigger: `text-sm text-steel-400 hover:text-steel-100` button — "Show {N} archived versions" / "Hide archived versions"
- Chevron icon (Lucide `ChevronDown` / `ChevronUp`) rotates 180deg on open
- Archived rows: same `VideoVersionRow` layout but actions limited to "Restore" and "Permanently delete"

## Interaction Contracts

### "Generate new version" button
- Position: top of `VideoGeneratePanel`, above format picker and version list
- Size: `h-[72px] w-full` (glove-friendly primary CTA)
- Color: `bg-brand-orange text-steel-900 font-semibold text-lg` (orange = action, not worker-facing publish)
- Disabled state when a generation is already active for the selected format: `opacity-50 cursor-not-allowed`
- On click: show format picker modal if no format selected, else trigger `generateNewVersion` action
- Label: "Generate new version" (not "Re-generate" — per D-06)

### Version row action icons
All icon buttons: 32px minimum tap target, `text-steel-400 hover:text-steel-100 transition-colors`.

| Icon | Lucide Component | Action | Condition |
|------|-----------------|--------|-----------|
| Edit label | `Edit2` (16px) | Open inline label editor | Always visible on ready/failed rows |
| Publish | `Upload` (16px) | Trigger publish confirmation | Only on ready + unpublished rows |
| Unpublish | `EyeOff` (16px) | Trigger unpublish (no confirmation — low consequence) | Only on published rows |
| Archive | `Archive` (16px) | Trigger archive confirmation | Ready/failed rows; disabled on active rows |
| Restore | `ArchiveRestore` (16px) | Unarchive immediately (no confirmation) | Archived rows only |
| Permanent delete | `Trash2` (16px) | Trigger permanent delete confirmation | Archived rows only |

### Inline label editor
- Trigger: clicking "Add label..." text OR the Edit2 icon
- Renders an `<input type="text">` in place of the label text
- Max length: 60 characters
- Placeholder: "E.g. Final cut, Training version..."
- Auto-focus on open
- Saves on: blur OR Enter key
- Cancels on: Escape key (restores previous value without saving)
- Visual: `bg-steel-900 border border-steel-600 rounded px-2 py-1 text-sm text-steel-100 w-full max-w-[240px]`

### Inline confirmation panels
Pattern matches existing `VideoAdminPreview` confirm pattern: renders inline below the row, `role="alertdialog"`.

| Action | Confirm copy | Confirm button | Cancel button |
|--------|-------------|---------------|---------------|
| Publish | "Publish this version? Workers will see it in the SOP video tab. Any currently published version will be unpublished." | "Yes, publish" (`bg-brand-yellow text-steel-900`) | "Not yet" (`bg-steel-700`) |
| Archive | "Archive this version? You can restore it from the archived section." | "Yes, archive" (`bg-steel-600 text-steel-100`) | "Keep version" (`bg-steel-700`) |
| Permanent delete | "Permanently delete this version? The video file and all data will be removed and cannot be recovered." | "Delete permanently" (`bg-red-600 text-steel-100`) | "Keep it" (`bg-steel-700`) |

All confirm buttons: `h-[44px] flex-1 font-semibold rounded-lg` (matches existing `VideoAdminPreview` pattern).

### Publish exclusivity feedback
When publishing version X, the UI must reflect that the previously published version is now unpublished. After the `publishVersionExclusive` action resolves:
- TanStack Query `invalidateQueries(['video-versions', sopId])` re-fetches all rows
- No separate toast — the list re-renders with the new published/unpublished states visually

### Archive collapsible
- Default state: collapsed
- Trigger text: "Show {N} archived version{s}" (pluralise correctly)
- No animation required — `hidden/block` toggle is sufficient

## States and Their Visual Representations

| Version status | Row visual |
|---------------|-----------|
| `queued` / `analyzing` / `generating_audio` / `rendering` | Blue pulsing badge + inline `VideoGenerationStatus` stepper expanded; archive icon disabled |
| `ready` + `published=true` | Left border `brand-yellow`, green "Published" badge, Unpublish + Archive icons |
| `ready` + `published=false` | No left border, neutral "Ready" badge, Publish + Archive icons |
| `failed` | Red "Failed" badge, Edit label + Archive icons; no Publish icon |
| `archived=true` | Greyed row (`text-steel-400`), Restore + Trash2 icons only; no Publish or Archive icons |
