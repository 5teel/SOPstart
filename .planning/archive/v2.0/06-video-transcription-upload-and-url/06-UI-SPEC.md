#### `AdversarialFlagBanner.tsx` (new — inline within right pane of ReviewClient)

**Purpose:** Surfaces discrepancies found by the adversarial verification pass.

**States:**
- `no-flags`: renders nothing
- `has-flags`: renders amber banner above section editors
- `dismissed`: renders nothing (user dismissed the banner)

**Layout (has-flags state):**
```
[AlertTriangle icon] AI verification found {n} potential issue{s} — review before publishing.
[Expand chevron]
  --- expanded list of flags ---
  [Flag 1]: {section name} — {discrepancy description}
  [Flag 2]: ...
  ---
[Dismiss — text-steel-400 text-xs]
```
- Banner container: `bg-brand-orange/20 border border-brand-orange/50 rounded-lg px-4 py-3 mb-4`
- Headline text: `text-sm font-semibold text-brand-orange`
- Flag list item: `text-xs text-steel-100 py-1`
- Dismissed via "Dismiss" link — `text-xs text-steel-400 hover:text-steel-100`; admin must explicitly dismiss (flags do not auto-clear)
- Flags do not block publish — warn-but-allow consistent with D-13 and Phase 5 approach

#### `MissingSectionWarningBanner.tsx` (new — inline within right pane of ReviewClient)

**Purpose:** Warns when hazards or PPE sections are absent from the video-sourced SOP (VID-07 / D-13).

**Layout:**
```
[AlertTriangle icon] Warning: No {hazards / PPE / hazards and PPE} section detected in this SOP.
Admin can acknowledge and publish anyway.
```
- Same amber banner style as `AdversarialFlagBanner`: `bg-brand-orange/20 border border-brand-orange/50 rounded-lg px-4 py-3 mb-4`
- "I understand — publish anyway" checkbox inline at bottom of banner: `flex items-center gap-2 mt-2 text-xs text-steel-400` — checkbox must be checked before "Publish SOP" header button activates
- If both hazards AND PPE are absent, show a single combined banner: "Warning: No hazards or PPE section detected..."
- Banner does not auto-dismiss; remains visible until checkbox checked

## Interaction Contract

### Upload Mode — Video File

| Trigger | Response |
|---------|----------|
| Drop or select MP4/MOV file | Add to queue with `Video` icon (purple-400), filename, file size, TUS progress bar (all video files use TUS — exceed 50MB threshold) |
| File queued | Show `TusUploadProgress` bar; no spinner (all videos use TUS path) |
| Upload in progress | Named stage row updates in `ParseJobStatus`; stepper advances: Uploading → Extracting → Transcribing → Structuring → Verifying |
| Any processing stage fails | Failure card shows failed stage name + brief error + "Retry from {stage}" + "Delete" links |
| "Retry from {stage}" clicked | Re-queues job from failed stage, not from scratch; stage stepper resets to failed stage |
| Processing complete | "Transcript and SOP ready to review" card with `CheckCircle`; "Review now →" link navigates to review page |
| Video file > 2GB | Toast: "{filename} is over 2GB. Please compress the video or split into shorter clips." |

### Upload Mode — YouTube URL

| Trigger | Response |
|---------|----------|
| User switches to "YouTube URL" tab | Dropzone replaced with URL input + terms checkbox + CTA button |
| URL input focused | No validation yet — validate on submit |
| CTA button clicked with empty URL | Inline validation: "Please enter a YouTube URL" below input field, `text-xs text-red-400` |
| CTA button clicked without terms checkbox | Inline validation: "Please confirm rights before proceeding" below checkbox, `text-xs text-red-400` |
| Valid YouTube URL + checkbox checked → CTA clicked | Disable button ("Fetching captions…"), call YouTube caption API; spinner replaces button text |
| Captions found | Navigate to review page with transcript populated from captions; no audio transcription needed |
| No captions found | Inline message: "No captions found for this video. Download the video and upload as MP4 for audio transcription." — `text-sm text-brand-orange` below CTA. Show "Upload as MP4 instead" link that switches back to file upload tab |
| Invalid YouTube URL | "That doesn't look like a YouTube URL. Check the link and try again." — `text-xs text-red-400` |
| Non-YouTube URL pasted (e.g. Vimeo) | "Only YouTube URLs are supported. Paste an MP4 file URL instead, or upload the video file directly." |

### Review Page — Transcript Panel Interactions

| Trigger | Response |
|---------|----------|
| Click transcript line | Highlight line with `brand-yellow` left border; jump video player to that timestamp |
| Video player timestamp changes (timeupdate event) | Auto-scroll transcript panel to matching line; apply active highlight |
| Transcript panel scrolled manually | Disable auto-scroll-follow until player timestamp changes by >5s |
| Click "Review now →" (from ParseJobStatus) | Navigate to review page, left pane shows `VideoReviewPanel`, right pane shows structured SOP sections |

### Review Page — Adversarial Flags Interactions

| Trigger | Response |
|---------|----------|
| Flags exist on page load | `AdversarialFlagBanner` renders collapsed with flag count |
| Click expand chevron | Flag list expands; each flag shows section + discrepancy |
| Click "Dismiss" | Banner collapses; flags logged but not blocking; no longer visible |
| Flags dismissed | Publish button enabled if sections approved (flags do not block publish) |

### Review Page — Missing Section Interactions

| Trigger | Response |
|---------|----------|
| Missing hazards or PPE section detected | `MissingSectionWarningBanner` renders above section editors |
| Admin checks "I understand — publish anyway" | Checkbox state enables publish flow; banner stays visible as reminder |
| Admin unchecks the checkbox | Publish button re-disables |

### Destructive Actions — Review Page

| Action | Approach |
|--------|----------|
| "Re-parse" (rebuild from transcript) | Inline confirm: "This will discard your edits and re-structure from the transcript. Sure?" — [Yes, re-parse] (`bg-brand-orange`) + [Cancel] (`bg-steel-700`) |
| "Delete draft" | Inline confirm: "This will permanently delete this draft. Can't be undone." — [Yes, delete] (`bg-red-600`) + [Cancel] (`bg-steel-700`) |
| "Publish SOP" | Inline confirm: "Publish this SOP? It will be visible to all workers." — [Yes, publish] (`bg-brand-yellow text-steel-900`) + [Cancel] (`bg-steel-700`) |

These use the existing inline-confirm pattern from `ReviewClient.tsx` — no modal dialogs.

## Accessibility Contract

- All `<button>` elements have explicit `aria-label` when icon-only (e.g., tab close, dismiss banner)
- `AdversarialFlagBanner` expand/collapse: `aria-expanded={isExpanded}` on chevron button
- `MissingSectionWarningBanner` checkbox: `htmlFor` label association; `aria-required="true"`
- Video player (`<video>`): include `aria-label="SOP source video"` on the element
- YouTube `<iframe>`: include `title="SOP source video"` and `allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture"` attributes
- Transcript panel: `role="list"` on container, `role="listitem"` on each line; `aria-pressed` on active line
- ProcessJobStatus stepper: each step wrapped with `aria-current="step"` on active step, `aria-label="{stage name}"` on each step element
- Terms acknowledgement checkbox: `required` attribute; inline error uses `role="alert"` on the validation message element
- Tab mode toggle: use `role="tab"` / `aria-selected` / `role="tabpanel"` ARIA pattern
