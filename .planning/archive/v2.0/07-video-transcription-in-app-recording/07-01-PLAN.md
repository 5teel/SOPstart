**VideoRecorder.tsx** (`src/components/admin/VideoRecorder.tsx`):

1. **Props:** `{ open: boolean; onClose: () => void; onSubmitComplete: (sopId: string) => void }`.

2. **State machine** with states: `requesting-permission`, `permission-denied`, `ready`, `recording`, `stopping`, `extracting-audio`, `preview`, `error`, `discard-confirm`. Use a single `useState<RecorderState>` where `RecorderState` is a union type defined in the file.

3. **Camera setup (requesting-permission -> ready):**
   - On mount when `open=true`, call `navigator.mediaDevices.getUserMedia({ video: { facingMode: 'environment' }, audio: true })`.
   - On success, attach stream to a `<video ref={videoRef}>` element with `autoPlay muted playsInline`.
   - On failure (NotAllowedError), transition to `permission-denied`.
   - Default camera is rear (`facingMode: 'environment'`) per D-07 / UI-SPEC discretion section.

4. **Switch camera button:**
   - Toggle `facingMode` between `'environment'` and `'user'`.
   - Stop existing tracks, re-acquire stream with new facingMode.
   - When facingMode is `'user'`, apply `style={{ transform: 'scaleX(-1)' }}` to mirror the viewfinder.
   - Only show on mobile: wrap with `className="md:hidden"`. On desktop, render a `w-[72px]` invisible spacer.

5. **MediaRecorder configuration (per UI-SPEC MediaRecorder Configuration section):**
   - Codec selection: iterate `['video/webm;codecs=vp9,opus', 'video/webm;codecs=vp8,opus', 'video/webm', 'video/mp4']` and pick first where `MediaRecorder.isTypeSupported()` returns true.
   - Options: `{ mimeType: selectedMime, videoBitsPerSecond: 2_500_000, audioBitsPerSecond: 128_000 }`.
   - Collect chunks in a `chunksRef = useRef<Blob[]>([])`.
   - `ondataavailable`: push `event.data` to chunksRef.
   - `onstop`: create `new Blob(chunksRef.current, { type: selectedMime })`, store in state as `recordedBlob`, transition to `extracting-audio`.

6. **Recording timer and 15-minute cap (D-06):**
   - Start a `setInterval(1000)` on recording start. Track `elapsedSeconds` in state.
   - Display as `MM:SS` format using `tabular-nums`.
   - At 13 * 60 (780s), switch timer to `text-red-400` and add `({remaining} left)` suffix.
   - At 15 * 60 (900s), auto-call `mediaRecorder.stop()`. Show toast: "Maximum recording length reached (15 minutes)."
   - Clean up interval on stop.

7. **Recording indicator:** Absolute positioned `top-4 left-4` over viewfinder. Red dot with `animate-pulse`, "REC" label `text-xs font-semibold text-steel-100`, elapsed timer `text-xs font-semibold text-steel-400 tabular-nums`. Visible only in `recording` state.

8. **Record/Stop button (center of controls bar):**
   - Idle: `w-20 h-20 rounded-full border-4 border-brand-yellow bg-steel-800` with inner white filled Circle icon (w-8 h-8). `aria-label="Start recording"`.
   - Recording: `w-20 h-20 rounded-full border-4 border-red-500 bg-red-500` with inner white Square icon (w-8 h-8). Outer pulse ring `ring-4 ring-red-500/30`. `aria-label="Stop recording"`.
   - Disabled during extraction: `opacity-50 cursor-not-allowed`.

9. **Audio extraction (extracting-audio state):**
   - Create a `File` from the recorded blob: `new File([recordedBlob], 'recording.webm', { type: selectedMime })`.
   - Call `extractAudioFromVideo(videoFile, onProgress)` from `@/lib/parsers/extract-video-audio`.
   - Show `TusUploadProgress` with extraction percentage + "Extracting audio... {pct}%" text below.
   - On completion, store `audioFile` in state and transition to `preview`.
   - On error, transition to `error` state with message "Audio extraction failed. Please try recording again."

10. **Discard confirmation (discard-confirm state):**
    - Triggered when user taps X close button while in `recording` state.
    - Replaces controls bar content with inline confirm per UI-SPEC Layout Contracts > Discard Confirmation.
    - "Stop recording and discard?" heading + "Your recording will be lost." sub-copy.
    - Two buttons: "Discard recording" (`text-red-400 border border-red-500/50`) and "Keep recording" (`bg-steel-700 text-steel-100`).
    - Discard: stop MediaRecorder, clear blob, stop all tracks, transition to closed (call `onClose()`).
    - Keep: transition back to `recording` state.

11. **Close button behavior:**
    - If `recording` state: transition to `discard-confirm`.
    - All other states: stop all media tracks, revoke object URLs, call `onClose()`.

12. **Cleanup on unmount:** Stop all media tracks, clear intervals, revoke object URLs.

13. **Layout:** Follow UI-SPEC VideoRecorder Overlay layout exactly:
    - Root: `fixed inset-0 z-50 bg-steel-900 flex flex-col` with `role="dialog" aria-modal="true" aria-label="Record video"`.
    - Header: `h-14 px-4 flex items-center justify-between bg-steel-900`.
    - Viewfinder: `flex-1 relative bg-steel-900 overflow-hidden`.
    - Controls: `bg-steel-800 py-6 px-8 flex items-center justify-between`.

14. **Accessibility:** All aria attributes per UI-SPEC Accessibility Contract section. Focus trap within overlay when open.

**VideoPreviewPanel.tsx** (`src/components/admin/VideoPreviewPanel.tsx`):

1. **Props:** `{ videoBlob: Blob; audioFile: File; onRetake: () => void; onClose: () => void; onSubmitComplete: (sopId: string) => void }`.

2. **Video playback:** Create object URL from `videoBlob` on mount (revoke on unmount). Render `<video src={objectUrl} controls playsInline className="w-full h-full object-contain" aria-label="Recording preview">`.

3. **Duration badge:** `absolute bottom-4 right-4 bg-steel-900/80 rounded px-2 py-1 text-xs font-semibold text-steel-100 tabular-nums`. Compute duration from video element's `loadedmetadata` event, format as `MM:SS`.

4. **States:** `reviewing`, `uploading`, `upload-error`. Track with `useState<'reviewing' | 'uploading' | 'upload-error'>('reviewing')`.

5. **Submit flow (per D-07 — audio-only upload):**
   - Set state to `uploading`. Disable both buttons.
   - Call `createVideoUploadSession({ name: 'recording.mp3', size: String(audioFile.size), type: 'audio/mpeg' })`.
   - If error, show toast and set state to `upload-error`.
   - On success, TUS upload the `audioFile` to `sop-videos` bucket using `tusUpload()`:
     - `file: audioFile`
     - `storagePath: sessionResult.path`
     - `accessToken: sessionResult.token`
     - `bucketName: 'sop-videos'`
     - `onProgress`: update a local progress state (show as disabled button text "Uploading... {pct}%").
     - `onSuccess`: trigger `/api/sops/transcribe` with `{ sopId: sessionResult.sopId }`, then call `onSubmitComplete(sessionResult.sopId)`.
     - `onError`: toast "Upload failed -- please check your connection and try again." Set state to `upload-error`.

6. **Retake:** Calls `onRetake()` which parent handles (destroy blob, restart camera).

7. **Layout:** Follow UI-SPEC VideoPreviewPanel layout exactly:
   - Root: `fixed inset-0 z-50 bg-steel-900 flex flex-col` with `role="dialog" aria-modal="true" aria-label="Review your recording"`.
   - Header with close (X), "Preview" label centered, spacer.
   - Playback area: `flex-1 relative bg-steel-900`.
   - Action bar: `bg-steel-800 py-6 px-6 flex gap-4`.

   - Submit: `flex-[2] h-[72px] bg-brand-yellow text-steel-900 font-semibold text-lg rounded-lg`.
