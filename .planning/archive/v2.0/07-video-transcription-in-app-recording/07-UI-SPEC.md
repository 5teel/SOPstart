## Interaction Contract

### Recording Flow — Capable Device

| Trigger | Response |
|---------|----------|
| Admin taps "Record video" tab | Switch to record tab panel. Check MediaRecorder support. Capable: show "Start recording" button. |
| Admin taps "Start recording" | Open `VideoRecorder` overlay. Begin requesting camera + microphone permissions. |
| Permission request pending | Show `Loader2` in viewfinder; record button disabled; copy "Requesting camera access…" below button |
| Permission granted | Start live preview stream. Record button activates (idle state). |
| Permission denied | `permission-denied` state: show `AlertTriangle` icon + "Camera access required — please allow camera access in your browser settings and reload." + "How to enable" link to MDN docs |
| Admin taps Record button (idle) | `MediaRecorder.start()`. Transition to `recording` state. Show red REC indicator + elapsed timer. Disable camera switch. |
| Admin taps Stop button (recording) | `MediaRecorder.stop()`. Transition to `stopping` state briefly. |
| Recording auto-stops at 15 minutes | Same as tapping Stop. Show toast: "Maximum recording length reached (15 minutes)." |
| Elapsed timer reaches 13:00 | Countdown warning: timer turns red; add "({2:XX} left)" suffix |
| Audio extraction begins | Transition to `extracting-audio` state. Show `TusUploadProgress` style bar for extraction progress (0–100%). Copy: "Extracting audio… {pct}%" |
| Audio extraction complete | Transition to `preview` state. Show `VideoPreviewPanel` with recorded video for review. |
| Audio extraction fails | `error` state: "Audio extraction failed. Please try recording again." + "Try again" button that resets to `ready` state |
| Admin taps "Retake" in preview | Destroy recorded blob. Close `VideoPreviewPanel`. Re-open `VideoRecorder` in `ready` state (camera preview restarted). |
| Admin taps "Submit for transcription" | Disable button. Call `createVideoUploadSession` server action with audio file metadata. TUS upload extracted audio. On success: trigger `/api/sops/transcribe`. Navigate to SOP review page. |
| TUS upload fails | Toast: "Upload failed — please check your connection and try again." Button re-enables. |
| Admin taps "Switch camera" | Enumerate `MediaDevices` and toggle between `facingMode: 'user'` and `facingMode: 'environment'`. Mirror viewfinder when front camera active. |
| Admin taps X (overlay close) | If in `recording` state: show inline confirm within overlay: "Stop recording and discard? Your recording will be lost." — [Discard recording] (text-red-400) + [Keep recording] (text-steel-100). If not recording: close immediately. |

### iOS Fallback Flow

| Trigger | Response |
|---------|----------|
| Admin taps "Record video" tab (iOS) | Detect no MediaRecorder. Show iOS fallback panel instead of "Start recording" button. |
| Admin taps "Choose video file" | Open native file picker with `accept="video/mp4,video/quicktime"`. Switch UploadDropzone to `upload` tab after file selected. File added to queue normally. |
| File selected and tab switched | Admin sees file queue in Upload tab with normal upload flow. |

### Permission Denied

| Trigger | Response |
|---------|----------|
| Camera permission denied by browser | `permission-denied` state with `AlertTriangle` icon |
| Admin taps "Open Settings" | Open browser-specific settings URL if available, otherwise show: "In your browser's address bar, tap the camera icon or go to Settings > Privacy > Camera and allow access for this site." — `text-xs text-steel-400 leading-relaxed mt-2` |

## MediaRecorder Configuration (Claude's Discretion)

These are implementation decisions within the discretion area from CONTEXT.md:

**Codec / MIME type selection order:**
1. `video/webm;codecs=vp9,opus` — best quality, Chrome/Android support
2. `video/webm;codecs=vp8,opus` — wider webm support
3. `video/webm` — generic webm fallback
4. `video/mp4` — Safari fallback (if supported)

Use `MediaRecorder.isTypeSupported()` to iterate and select first supported type.

**Bitrate:** `videoBitsPerSecond: 2_500_000` (2.5 Mbps) — sufficient quality for reading safety text; `audioBitsPerSecond: 128_000`.

**Camera selection:**
- Default: `facingMode: 'environment'` (rear camera) — filming a procedure, not the recorder
- Switch camera: toggle between `'environment'` and `'user'`; re-acquire stream on switch

**Recording indicator animation:** CSS `animate-pulse` on the red dot (Tailwind built-in, no custom keyframes needed).

**Pause/Resume:** Not implemented in Phase 7 (deferred per CONTEXT.md Deferred Ideas). Record and Stop only.
