    1. Create `src/lib/validators/auth.ts` with Zod schemas:
       ```typescript
       import { z } from 'zod'

       export const orgSignUpSchema = z.object({
         organisationName: z.string().min(2, 'Organisation name must be at least 2 characters').max(100),
         email: z.string().email('Enter a valid email address'),
         password: z.string().min(8, 'Password must be at least 8 characters'),
         confirmPassword: z.string(),
       }).refine(data => data.password === data.confirmPassword, {
         message: 'Passwords do not match',
         path: ['confirmPassword'],
       })

       export const loginSchema = z.object({
         email: z.string().email('Enter a valid email address'),
         password: z.string().min(1, 'Password is required'),
       })

       export const inviteCodeSchema = z.object({
         code: z.string().min(1, 'Invite code is required').max(20).transform(v => v.toUpperCase().trim()),
       })

       export const inviteWorkerSchema = z.object({
         email: z.string().email('Enter a valid email address'),
       })

       export const acceptInviteSchema = z.object({
         password: z.string().min(8, 'Password must be at least 8 characters'),
         confirmPassword: z.string(),
         token: z.string().min(1, 'Invite token is required'),
       }).refine(data => data.password === data.confirmPassword, {
         message: 'Passwords do not match',
         path: ['confirmPassword'],
       })

       export const updateRoleSchema = z.object({
         memberId: z.string().uuid(),
         role: z.enum(['worker', 'supervisor', 'admin', 'safety_manager']),
       })

       export type OrgSignUpInput = z.infer<typeof orgSignUpSchema>
       export type LoginInput = z.infer<typeof loginSchema>
       export type InviteCodeInput = z.infer<typeof inviteCodeSchema>
       export type InviteWorkerInput = z.infer<typeof inviteWorkerSchema>
       export type AcceptInviteInput = z.infer<typeof acceptInviteSchema>
       export type UpdateRoleInput = z.infer<typeof updateRoleSchema>
       ```

    2. Create `src/actions/auth.ts` with Server Actions for all auth flows:

       **signUpOrganisation:** (D-01, D-02, D-03, AUTH-01)
       - Validate input with `orgSignUpSchema`
       - Use `createAdminClient()` (service_role) to: create the org in `organisations` table, sign up the user with `supabase.auth.admin.createUser({ email, password, email_confirm: true })`, insert the user into `organisation_members` with `role: 'admin'`
       - Redirect to `/login` with success message

       **loginWithEmail:** (AUTH-02, AUTH-03)
       - Validate with `loginSchema`
       - Use server-side `createClient()` (from `@/lib/supabase/server`)
       - Call `supabase.auth.signInWithPassword({ email, password })`
       - On success, redirect to `/dashboard`
       - On error, return `{ error: 'Invalid email or password' }`

       **joinWithInviteCode:** (D-07, AUTH-02)
       - Validate with `inviteCodeSchema`
       - Use `createClient()` to get current user via `supabase.auth.getUser()`
       - CRITICAL: Check if user already has a row in `organisation_members`. If yes, return error: "You are already a member of an organisation." (Pitfall 3 from research)
       - Look up org by `invite_code` in `organisations` table
       - If not found, return error: "Invalid invite code"
       - Insert into `organisation_members` with `role: 'worker'`
       - Call `supabase.auth.refreshSession()` to get updated JWT claims
       - Redirect to `/dashboard`

       **inviteWorker:** (D-05, D-06, AUTH-02)
       - Validate with `inviteWorkerSchema`
       - Get current user's org_id from JWT claims
       - Use `createAdminClient()` to call `supabase.auth.admin.inviteUserByEmail(email, { data: { organisation_id, invited_role: 'worker' }, redirectTo: '/invite/accept' })`
       - Return success or error message

       **acceptInvite:** (D-06, AUTH-02)
       - Validate with `acceptInviteSchema`
       - This server action handles the second half of the email invite flow — after the worker clicks the invite link and lands on `/invite/accept`
       - Use `createClient()` (server-side) to call `supabase.auth.verifyOtp({ token_hash: token, type: 'invite' })` to exchange the invite token for a session
       - On successful verification, the user's session is established
       - Read the `organisation_id` and `invited_role` from the user's `user_metadata` (set by `inviteWorker` via the `data` param)
       - Insert into `organisation_members` with the `organisation_id` from metadata and `role: 'worker'` (or the invited_role from metadata)
       - Call `supabase.auth.refreshSession()` to get updated JWT with org claims
       - If the user needs to set a password (first login via invite), call `supabase.auth.updateUser({ password })` with the password from the form
       - Redirect to `/dashboard`
       - On error (invalid/expired token), return `{ error: 'This invite link is invalid or has expired. Please ask your admin to send a new invite.' }`

       **updateMemberRole:** (D-10 through D-15, AUTH-04)
       - Validate with `updateRoleSchema`
       - Use server-side `createClient()` (RLS enforced — only admins can update per RLS policy)
       - Update the member's role in `organisation_members`
       - Return success or error
