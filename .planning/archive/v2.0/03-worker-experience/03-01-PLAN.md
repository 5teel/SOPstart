**1. Create `src/lib/offline/sync-engine.ts`:**
- Import `db` from `@/lib/offline/db`
- Import `SupabaseClient` from `@supabase/supabase-js`
- Export `async function syncAssignedSops(supabase: SupabaseClient): Promise<{ synced: number; errors: string[] }>`
- Implementation:
  1. Fetch manifest from Supabase: `supabase.from('sop_assignments').select('sop_id, sops(id, version, updated_at)')` — RLS auto-filters to the current user's assignments
  2. Get cached versions from Dexie: `db.sops.toArray()` then build `Map<sopId, version>`
  3. Compute stale IDs: where manifest version differs from cached version or sopId not in cache
  4. If no stale IDs, return `{ synced: 0, errors: [] }`
  5. Fetch full SOP content for stale IDs: `supabase.from('sops').select('*, sop_sections(*, sop_steps(*), sop_images(*))').in('id', staleIds)`
  6. Write to Dexie in a single transaction (`db.transaction('rw', [db.sops, db.sections, db.steps, db.images], async () => {...})`):
     - For each SOP: `db.sops.put({ ...sop, _cachedAt: Date.now() })` (strip nested relations before put)
     - For each section: `db.sections.put(section)` (strip nested steps/images)
     - For each step: `db.steps.put(step)`
     - For each image: `db.images.put(image)`
  7. Remove SOPs from Dexie that are no longer assigned: get all cached sopIds, subtract manifest sopIds, bulk delete orphans
  8. Update `syncMeta`: `db.syncMeta.put({ key: 'lastSync', value: new Date().toISOString() })`
  9. Return `{ synced: staleIds.length, errors: [] }`
  10. Wrap entire function in try/catch — on error, return `{ synced: 0, errors: [error.message] }`
