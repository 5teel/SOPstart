    Three TanStack Query hooks:

    a. `useWorkerCompletions()`:
    - queryKey: ['completions', 'worker']
    - Fetches from Supabase `sop_completions` with `.select('*, sops(title)')` ordered by `submitted_at` desc, limit 50
    - RLS automatically scopes to worker's own completions
    - Also fetch any sign-off records: join `completion_sign_offs` via `completion_id`
    - networkMode: 'offlineFirst'
    - Returns typed array with sop title, status, submitted_at, photo count (from completion_photos count)

    b. `useSupervisorCompletions(filter: { type: 'all' | 'by_sop' | 'by_worker', value?: string })`:
    - queryKey: ['completions', 'supervisor', filter.type, filter.value]
    - Fetches sop_completions with `.select('*, sops(title), completion_sign_offs(*)')` ordered by submitted_at desc
    - RLS handles scoping (supervisor sees only their supervised workers, safety_manager sees all)
    - When filter.type === 'by_sop': add `.eq('sop_id', filter.value)`
    - When filter.type === 'by_worker': add `.eq('worker_id', filter.value)`
    - Also need worker profile info: join to `auth.users` is not possible via RLS, so query `organisation_members` to get worker names. Use a separate query or embed worker display name in the completion data. Simplest: query `organisation_members` for display_name/email in a separate `useQuery` with the distinct worker_ids from the completions.
    - Include photo count per completion via a count subquery or separate query
    - networkMode: 'offlineFirst'

    c. `useCompletionDetail(completionId: string)`:
    - queryKey: ['completions', 'detail', completionId]
    - Fetches single sop_completion with steps, photos, sign-off record, SOP title, worker info
    - Select: `*, sops(title, version), completion_photos(id, step_id, storage_path, content_type), completion_sign_offs(id, supervisor_id, decision, reason, created_at)`

    **2. Create src/app/(protected)/activity/page.tsx:**

    Server component that role-branches:
    ```typescript
    export default async function ActivityPage() {
      const supabase = await createClient()
      const { data: { user } } = await supabase.auth.getUser()
      if (!user) redirect('/login')
      const { data: member } = await supabase.from('organisation_members')
        .select('role').eq('user_id', user.id).maybeSingle()
      const role = member?.role ?? null
      if (role === 'worker') return <WorkerActivityView />
      if (role === 'supervisor' || role === 'safety_manager') return <SupervisorActivityView role={role} />
      redirect('/dashboard')
    }
    ```
