**src/hooks/useVideoGeneration.ts** — Hook to fetch published video for a SOP:

Export `useVideoGeneration(sopId: string)` returning `{ data: VideoGenerationJob | null, isLoading: boolean }`.

Use TanStack Query (`useQuery`) with:
- queryKey: `['video-generation', sopId]`
- queryFn: Supabase client query to `video_generation_jobs` table, filter by `sop_id = sopId`, `published = true`, `status = 'ready'`, order by `created_at desc`, limit 1
- enabled: only when `sopId` is truthy AND online (use `useOnlineStatus()` — video data is online-only per Pitfall 7)
- staleTime: 5 minutes (video availability doesn't change frequently)
- No persister (online-only data must NOT be stored in Dexie/IndexedDB)

The hook fetches video availability from Supabase directly (NOT bundled into the SOP sync that writes to Dexie). This prevents video URLs from leaking into offline storage per Pitfall 7.

matcher({ url }: { url: URL }) {
