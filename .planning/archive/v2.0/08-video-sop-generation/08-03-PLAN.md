**src/app/(protected)/admin/sops/[sopId]/video/page.tsx** — Admin video generation page route:

Server component that:
1. Fetches the SOP record (title, status, updated_at, version) via admin Supabase client
2. Fetches the latest `video_generation_jobs` record for this SOP (any format, ordered by created_at desc) via admin client
3. Passes SOP data and existing job (if any) as props to `VideoGeneratePanel`
4. Redirects to `/admin/sops` if SOP not found or not published

**src/components/admin/VideoGenerationStatus.tsx** — Client component:

Props: `{ jobId: string, initialStatus?: VideoGenStatus, initialStage?: string, onComplete?: (job: VideoGenerationJob) => void, onFailed?: (error: string) => void }`

5-step horizontal stage stepper (per UI-SPEC):
```
[Analysing] -- [Slides] -- [Narration] -- [Rendering] -- [Ready]
```

Stage mapping:
- `analyzing` -> step 1 active
- `generating_audio` -> step 2 active (label: "Slides" for slideshow, "Content" for screen-recording)
- Stage between generating_audio and rendering -> step 3 active "Narration"
- `rendering` -> step 4 active
- `ready` -> all steps completed (green checkmarks)
- `failed` -> failed step highlighted

Layout: `flex items-center gap-1 mb-4 overflow-x-auto`
Step label: `text-xs whitespace-nowrap px-1`
Connecting line: `h-px flex-1 bg-steel-700 min-w-[8px]`, completed: `bg-brand-yellow`
Active step: `text-brand-yellow font-semibold`
Completed step: `text-green-400`
Pending step: `text-steel-600`

**Supabase Realtime subscription** (same pattern as ParseJobStatus.tsx):
- Subscribe to `postgres_changes` on `video_generation_jobs` table, filter `id=eq.{jobId}`
- On payload: update local status and stage state
- 5-second setTimeout fallback: if no Realtime event within 5s after mount, query `video_generation_jobs` directly via the Supabase browser client filtered by `id = jobId` (use `supabase.from('video_generation_jobs').select('*').eq('id', jobId).single()`). Do NOT fetch a GET API route — no GET handler exists at `/api/sops/generate-video/{jobId}`. Continue polling via direct Supabase query every 5 seconds until status reaches a terminal state ('ready' or 'failed').
- When status reaches `ready`: call `onComplete` callback with full job data
- When status reaches `failed`: call `onFailed` callback with error_message

Status card container: `bg-steel-800 border border-steel-700 rounded-lg p-4`
In-progress: Loader2 spinner (`text-blue-400 animate-spin`) + stage copy (e.g., "Analysing SOP structure...")
Ready: CheckCircle (`text-green-400 size-20`) + "Video ready to preview and publish"
Failed: AlertTriangle (`text-brand-orange size-20`) + "Generation failed - {error}"
